## 1.1 类型修饰与转换关键词

### const

**概念**：const 是 "constant" 的缩写，用于定义常量或声明对象不可修改。

**底层原理**：
- const 对象通常被放入**只读存储区**（.rodata），编译器可以进行优化
- const 成员变量必须在**初始化列表**中初始化
- const 成员函数承诺不修改对象的非 mutable 成员

**面试重点**：

```cpp
// 1. const 与指针的组合
const int* p1;      // p1 是指针，指向的 int 是 const（底层 const）
int* const p2;     // p2 是 const 指针，指向的 int 可变（顶层 const）
const int* const p3; // 两者都是 const

// 2. const 与引用
const int& r = 42;  // 可以绑定字面量，延长临时对象生命周期
// int& r2 = 42;    // 错误！非 const 引用不能绑定临时对象

// 3. const 成员函数
class MyClass {
    int data;
public:
    int getData() const { return data; }  // const 成员函数
    // void setData(int v) const { data = v; }  // 错误！不能修改成员
};

// 4. mutable 与 const
class Widget {
    mutable int cache = 0;  // mutable 即使在 const 成员函数中也可修改
    mutable mutable bool cacheValid = false;
public:
    int compute() const {
        if (!cacheValid) {
            cache = expensiveComputation();  // 合法
            cacheValid = true;
        }
        return cache;
    }
};

// 5. const 正确性 - 好的设计应该让 const 正确传播
// 好的设计：参数尽可能用 const 引用
void process(const std::string& str);  // 高效且安全

// 6. const vs #define
const int MAX_SIZE = 100;  // 有类型检查、作用域限制
// #define MAX_SIZE 100   // 无类型、全局替换、容易出错
```

**const 与编译器优化**：
```cpp
void foo() {
    const int x = 42;
    int y = x + 10;  // 编译器可直接优化为 int y = 52;
    // x = 10;       // 编译错误
}
```

**const_cast 的使用**：
```cpp
void print(const char* str) {
    // const_cast 用于移除 const（但要确保原对象非 const）
    char* p = const_cast<char*>(str);
    p[0] = 'H';  // 危险！如果 str 原本就是 const，这是未定义行为
}

// 合法使用场景
void processBuffer(char* buffer, int size);
void processData(const char* data, int size) {
    processBuffer(const_cast<char*>(data), size);  // 确保 data 非const时合法
}
```

### volatile

**概念**：volatile 告诉编译器该变量可能被程序外部因素（如硬件、多线程）修改，禁止编译器进行某些优化。

**底层原理**：
- 每次访问 volatile 变量都必须从**内存读取**，不能使用寄存器缓存
- volatile 变量的读写操作不会被重排序到前后（但不是完全的内存屏障）

**面试重点**：

```cpp
// 1. volatile 的基本用法
volatile int flag = 0;

// 线程1
while (!flag) {
    // 等待，每次都从内存读取 flag
}

// 线程2
flag = 1;  // 设置标志

// 2. volatile 不能保证原子性！
volatile int counter = 0;
// counter++;  // 不是原子操作！需要 读取-修改-写入 三步
// 正确做法：使用 std::atomic<int>

// 3. volatile 与多线程
// volatile 不能替代锁或原子操作！
// 错误示例：
class BadSpinlock {
    volatile bool locked = false;
public:
    void lock() {
        while (locked);        // 忙等待
        locked = true;         // 竞争条件！
    }
    void unlock() {
        locked = false;
    }
};

// 正确做法：
class GoodSpinlock {
    std::atomic<bool> locked{false};
public:
    void lock() {
        while (locked.exchange(true, std::memory_order_acquire)) {
            // 忙等待
        }
    }
    void unlock() {
        locked.store(false, std::memory_order_release);
    }
};

// 4. volatile 的正确使用场景：内存映射 I/O
volatile uint32_t* const GPIO_REGISTER = reinterpret_cast<volatile uint32_t*>(0x40020000);
*GPIO_REGISTER = 0x01;  // 每次都直接操作硬件寄存器

// 5. const volatile 组合
const volatile uint32_t* STATUS_REG = /* 地址 */;
// const：程序不应修改
// volatile：值可能被硬件改变，每次都要重新读取
```

**volatile vs atomic**：

| 特性 | volatile | std::atomic |
|------|----------|-------------|
| 防止编译器优化 | ✓ | ✓ |
| 保证操作原子性 | ✗ | ✓ |
| 内存序保证 | ✗ | ✓ |
| 适用场景 | 硬件寄存器 | 多线程同步 |

### mutable

**概念**：mutable 允许 const 成员函数修改被其修饰的成员变量。

**使用场景**：
- **缓存/懒加载**：在 const 函数中实现缓存
- **引用计数**：在 const 函数中修改引用计数
- **互斥锁**：在 const 函数中加锁（锁本身需要修改状态）

**示例**：

```cpp
class Database {
    mutable std::mutex mutex;  // const 函数中也需要加锁
    mutable std::optional<std::string> cache;
    std::string data;

public:
    const std::string& getData() const {
        std::lock_guard lock(mutex);  // mutable 使其合法
        if (!cache) {
            cache = fetchFromDB();     // mutable 使其合法
        }
        return *cache;
    }
};
```

## 1.2 类型转换关键词

### const_cast

**用途**：添加或移除 const/volatile 修饰符。

**唯一合法的场景**：修改一个**原本非 const** 但通过 const 引用/指针访问的对象。

```cpp
// 1. 移除 const
void modify(const int* pci) {
    int* pi = const_cast<int*>(pci);
    *pi = 10;  // 只有当 pci 指向的对象本身非 const 时才是合法的
}

// 2. 添加 const（很少需要）
int* p = &x;
const int* pc = const_cast<const int*>(p);

// 3. 实际应用：重载 const 和非 const 版本
class Container {
public:
    const int& operator[](size_t index) const {
        return data[index];
    }

    int& operator[](size_t index) {
        // 通过 const_cast 复用 const 版本，避免代码重复
        return const_cast<int&>(static_cast<const Container&>(*this)[index]);
    }

private:
    std::vector<int> data;
};
```

### static_cast

**用途**：相关类型之间的转换，编译时进行检查。

**使用场景**：

```cpp
// 1. 基本类型转换
double d = 3.14;
int i = static_cast<int>(d);  // i = 3

// 2. 指针向上转型（derived 到 base）- 安全
class Base {};
class Derived : public Base {};
Derived* d = new Derived;
Base* b = static_cast<Base*>(d);  // 安全

// 3. 指针向下转型（base 到 derived）- 需要确保对象实际类型正确
Base* b = new Derived;
Derived* d = static_cast<Derived*>(b);  // 危险！需要确定 b 确实指向 Derived

// 4. void* 转换
void* v = malloc(sizeof(int));
int* p = static_cast<int*>(v);

// 5. 构造函数和转换运算符
class Widget {
public:
    explicit Widget(int);  // explicit 禁止隐式转换
    operator int() const;
};

Widget w(42);
int n = static_cast<int>(w);  // 调用 operator int()
```

### dynamic_cast

**用途**：安全的向下转型和多态类型转换，运行时检查。

**底层原理**：
- 需要**虚函数表**支持
- 使用 RTTI（Run-Time Type Information）
- 成功返回目标类型指针，失败返回 nullptr（指针）/ 抛出异常（引用）

```cpp
// 1. 向下转型
class Base {
public:
    virtual ~Base() = default;  // 需要 virtual
};

class Derived : public Base {
    void derivedMethod() {}
};

Base* basePtr = new Derived;

// 安全的向下转型
if (Derived* derivedPtr = dynamic_cast<Derived*>(basePtr)) {
    derivedPtr->derivedMethod();  // 转型成功
}

// 2. 交叉转型（cross-cast）
class A { public: virtual ~A() = default; };
class B { public: virtual ~B() = default; };
class C : public A, public B {};

A* a = new C;
B* b = dynamic_cast<B*>(a);  // 交叉转型成功

// 3. 引用版本的 dynamic_cast
try {
    Derived& d = dynamic_cast<Derived&>(*basePtr);
    d.derivedMethod();
} catch (const std::bad_cast& e) {
    std::cout << "转型失败: " << e.what() << std::endl;
}

// 4. dynamic_cast 的性能考虑
// dynamic_cast 需要遍历继承树，比 static_cast 慢
// 在性能关键代码中避免使用
```

**dynamic_cast vs static_cast**：

| 特性 | dynamic_cast | static_cast |
|------|--------------|-------------|
| 检查时机 | 运行时 | 编译时 |
| 安全性 | 安全（失败返回 nullptr） | 不安全（未定义行为） |
| 需要虚函数 | 是 | 否 |
| 性能 | 较慢 | 快 |

### reinterpret_cast

**用途**：底层的位模式重新解释，最危险的转换。

**使用场景**（极少使用）：
- 与 C 代码交互
- 处理硬件寄存器
- 序列化/反序列化

```cpp
// 1. 指针类型转换
int* p = new int(0x12345678);
char* cp = reinterpret_cast<char*>(p);
// 现在可以通过 cp 访问 int 的字节表示

// 2. 指针和整数之间的转换
uintptr_t addr = reinterpret_cast<uintptr_t>(p);
int* p2 = reinterpret_cast<int*>(addr);

// 3. 函数指针转换
void (*funcPtr)();
using FuncType = void(*)();
FuncType ft = reinterpret_cast<FuncType>(funcPtr);

// 4. 实际应用：检查字节序
bool isLittleEndian() {
    uint32_t x = 1;
    return *reinterpret_cast<uint8_t*>(&x) == 1;
}

// 5. 实际应用：类型双关（type punching）
// 不推荐，应使用 union 或 memcpy
float f = 3.14f;
uint32_t bits = *reinterpret_cast<uint32_t*>(&f);  // 访问 float 的位表示
```

**警告**：reinterpret_cast 是**平台相关**的，可能导致：
- 对齐问题
- 字节序问题
- 严格的别名规则违规

## 1.3 存储类关键词

### static

**概念**：static 有多种含义，取决于上下文。

**三种用法**：

```cpp
// 1. 静态局部变量 - 函数内保持状态
void counter() {
    static int count = 0;  // 只初始化一次
    count++;
    std::cout << count << std::endl;
}

// 2. 静态全局变量/函数 - 文件作用域（内部链接）
// file1.cpp
static int fileLocalVar = 0;  // 只在本文件可见
static void fileLocalFunc() {} // 只在本文件可见

// 3. 类静态成员 - 所有对象共享
class MyClass {
    static int count;  // 声明
public:
    MyClass() { count++; }
    static int getCount() { return count; }  // 静态成员函数
};

int MyClass::count = 0;  // 定义（必须在某个 .cpp 文件中）

// C++17: inline static 成员变量可以直接在类内定义
class MyClass2 {
    inline static int count = 0;  // C++17
};
```

**静态局部变量的初始化**：
```cpp
// C++11 之前：多线程不安全
void func() {
    static std::string str = expensiveInit();  // 可能多次初始化
}

// C++11 之后：保证线程安全的初始化
void func() {
    static std::string str = expensiveInit();  // 线程安全
    // 编译器生成类似以下的代码：
    // if (!initialized) {
    //     lock_guard lock(mtx);
    //     if (!initialized) {
    //         construct str;
    //         initialized = true;
    //     }
    // }
}
```

**静态成员的线程安全**：
```cpp
class AtomicCounter {
    static std::atomic<int> count;
public:
    static void increment() {
        count.fetch_add(1, std::memory_order_relaxed);
    }
};
```

### extern

**概念**：extern 声明变量/函数在其他翻译单元中定义。

**用法**：

```cpp
// 1. 声明外部变量
// file1.cpp
int globalVar = 42;

// file2.cpp
extern int globalVar;  // 声明（不分配内存）
void use() {
    globalVar = 10;    // 使用 file1.cpp 中的变量
}

// 2. extern "C" - C 链接约定
#ifdef __cplusplus
extern "C" {
#endif

    void c_function(int x);  // 使用 C 的命名规则
    int c_variable;

#ifdef __cplusplus
}
#endif

// 3. extern 模板 - 显式实例化声明
// template instantiation
// mytemplate.h
template <typename T>
T max(T a, T b) { return a > b ? a : b; }

// mytemplate.cpp
extern template int max<int>(int, int);  // 声明：不在当前 TU 实例化
template int max<int>(int, int);        // 显式实例化

// main.cpp
extern template int max<int>(int, int);  // 声明：使用时不再实例化
int main() {
    max(1, 2);  // 使用其他编译单元的实例化
}

// 4. constexpr 与 extern 变量
// C++17: inline 变量解决了 ODR 问题
class MyClass {
    static constexpr int value = 42;  // 声明
    // C++17: inline static constexpr int value = 42;  // 定义
};
// constexpr int MyClass::value;  // 需要（或使用 inline）
```

### thread_local

**概念**：thread_local 声明线程本地存储（TLS），每个线程有独立副本。

**用法**：

```cpp
// 1. 线程局部变量
thread_local int threadCounter = 0;

void increment() {
    threadCounter++;  // 每个线程有自己的 counter
}

// 2. 线程局部对象
thread_local std::string threadName;

void setThreadName(const std::string& name) {
    threadName = name;  // 每个线程有自己的 threadName
}

// 3. 实际应用：获取当前线程的随机数生成器
thread_local std::mt19937 rng{std::random_device{}()};
int getRandom() {
    std::uniform_int_distribution<int> dist(1, 100);
    return dist(rng);  // 每个线程有自己的生成器，无锁
}

// 4. 线程局部静态变量
void* getThreadLocalBuffer() {
    thread_local void* buffer = malloc(1024);  // 每个线程只分配一次
    return buffer;
}

// 5. 类静态 thread_local 成员
class ThreadCache {
    thread_local static std::vector<int> cache;
public:
    static std::vector<int>& getCache() { return cache; }
};
thread_local std::vector<int> ThreadCache::cache;  // 定义
```

**thread_local 的底层实现**：
- 使用线程本地存储（TLS）段
- 访问通过 FS/GS 段寄存器（x86-64）
- 比全局变量访问稍慢，但比锁快得多

### inline

**概念**：inline 建议编译器内联展开函数调用。

**现代含义**：
- **内联提示**：建议编译器内联（编译器可忽略）
- **ODR 豁免**：允许在多个翻译单元中定义相同函数
- **隐式 inline**：类内定义的成员函数默认 inline

```cpp
// 1. 显式 inline
inline int add(int a, int b) {
    return a + b;
}

// 2. 类内定义 - 隐式 inline
class MyClass {
    int getValue() const { return value; }  // 隐式 inline
    int value;
};

// 3. 类外定义 - 需要显式 inline（如果在头文件中）
class MyClass {
    int getValue() const;
};

inline int MyClass::getValue() const {  // 必须 inline，否则 ODR 违规
    return value;
}

// 4. inline 变量 (C++17)
inline int globalCounter = 0;  // 可以在多个头文件中定义

// 5. inline 与 constexpr
// constexpr 函数隐式 inline
constexpr int square(int n) {  // 隐式 inline
    return n * n;
}

// 6. 强制内联（编译器扩展）
// GCC/Clang
__attribute__((always_inline)) int criticalFunction() { return 0; }

// MSVC
__forceinline int criticalFunction() { return 0; }
```

**inline 的权衡**：
- **优点**：消除函数调用开销，支持头文件实现
- **缺点**：代码膨胀，可能影响缓存
- **编译器决定**：现代编译器会根据函数大小、调用频率等自动决定

## 1.4 访问控制关键词

### public / protected / private

**概念**：控制类成员的访问权限。

**访问级别**：

| 访问符 | 类内 | 派生类 | 外部 | 友元 |
|--------|------|--------|------|------|
| public | ✓ | ✓ | ✓ | ✓ |
| protected | ✓ | ✓ | ✗ | ✓ |
| private | ✓ | ✗ | ✗ | ✓ |

```cpp
class Base {
public:     // 公共接口
    void publicMethod() {}

protected:  // 派生类可访问
    void protectedMethod() {}

private:    // 仅类内和友元可访问
    void privateMethod() {}

    // 友元声明
    friend class FriendClass;
    friend void friendFunction(Base&);
};

class Derived : public Base {
    void useBase() {
        publicMethod();      // ✓
        protectedMethod();   // ✓
        // privateMethod();  // ✗ 编译错误
    }
};

class FriendClass {
    void access(Base& b) {
        b.publicMethod();    // ✓
        b.protectedMethod(); // ✓
        b.privateMethod();   // ✓（友元）
    }
};
```

**继承访问控制详解**：

继承方式决定了基类成员在派生类中的访问权限。规则如下：

| 基类成员 | public 继承 | protected 继承 | private 继承 |
|----------|-------------|-----------------|--------------|
| public 成员 | → public | → protected | → private |
| protected 成员 | → protected | → protected | → private |
| private 成员 | 不可访问 | 不可访问 | 不可访问 |

```cpp
class Base {
public:
    int pub = 1;
protected:
    int prot = 2;
private:
    int priv = 3;
};

// public 继承：IS-A 关系（最常见的继承）
class PublicDerived : public Base {
    void demo() {
        pub;   // ✓ 访问 public (仍然是 public)
        prot;  // ✓ 访问 protected
        // priv; // ✗ 不可访问
    }
};

// protected 继承：IMPLEMENTED-IN-TERMS-OF 关系
class ProtectedDerived : protected Base {
    void demo() {
        pub;   // ✓ 访问 protected (原 public 变为 protected)
        prot;  // ✓ 访问 protected
    }
};

// private 继承：HAS-A 关系（类似组合）
class PrivateDerived : private Base {
    void demo() {
        pub;   // ✓ 访问 private (原 public 变为 private)
        prot;  // ✓ 访问 private (原 protected 变为 private)
    }
};

// 外部访问
void test() {
    PublicDerived pd;
    pd.pub;   // ✓ public 成员可访问
    // pd.prot;  // ✗ protected 不可访问

    ProtectedDerived ptd;
    // ptd.pub;   // ✗ 原 public 变为 protected，外部不可访问

    PrivateDerived pvd;
    // pvd.pub;   // ✗ 原 public 变为 private，外部不可访问
}
```

**三种继承方式的使用场景**：

```cpp
// 1. public 继承 - IS-A 关系（子类是父类的一种）
class Vehicle {
public:
    void move() { std::cout << "Moving\n"; }
};

class Car : public Vehicle {
    // Car IS-A Vehicle
    // 可以在任何需要 Vehicle 的地方使用 Car
};

void drive(Vehicle& v) {
    v.move();
}

Car c;
drive(c);  // ✓ Car 可以替代 Vehicle

// -------------------------------------------------------

// 2. private 继承 - HAS-A 关系（实现细节）
class Engine {
public:
    void start() { std::cout << "Engine started\n"; }
};

// 使用 private 继承：Car HAS-A Engine（但不对外暴露 Engine 接口）
class Car2 : private Engine {
public:
    void start() {
        Engine::start();  // 内部使用，但外部不能通过 Car2 调用 Engine 接口
        std::cout << "Car starting\n";
    }
};

Car2 c2;
// c2.Engine::start();  // ✗ 编译错误：Engine 的接口是 private 的
c2.start();  // ✓ 只能调用 Car2 自己的接口

// 注意：private 继承通常可以用组合替代（推荐）
class Car3 {
    Engine engine;  // 组合（更清晰）
public:
    void start() {
        engine.start();
    }
};

// -------------------------------------------------------

// 3. protected 继承 - IMPLEMENTED-IN-TERMS-OF 关系
// 很少使用，主要场景：派生类的进一步派生需要访问基类接口
class Base {
public:
    void interface() {}
};

class Middle : protected Base {
    // Base 的 public 接口变为 protected
    // Middle 的派生类可以访问，但外部不能
};

class DeepDerived : public Middle {
    void demo() {
        interface();  // ✓ 可以访问（来自 protected 继承）
    }
};

Middle m;
// m.interface();  // ✗ 外部不能访问
```

**using 声明恢复访问权限**：

```cpp
class Base {
public:
    void publicFunc() {}
protected:
    void protectedFunc() {}
private:
    void privateFunc() {}
};

class PrivateDerived : private Base {
public:
    // 恢复特定成员的访问权限
    using Base::publicFunc;  // publicFunc 在 PrivateDerived 中变为 public
};

PrivateDerived pd;
pd.publicFunc();  // ✓ 通过 using 恢复为 public
// pd.protectedFunc();  // ✗ 仍然是 private
```

**继承方式总结**：

| 继承方式 | 含义 | 使用场景 | 是否是多态 |
|----------|------|----------|------------|
| public | IS-A | 子类是父类的一种 | ✓（虚函数） |
| protected | IMPLEMENTED-IN-TERMS-OF | 实现细节，但派生类需要访问 | ✗ |
| private | HAS-A（实现复用） | 实现细节，完全不对外暴露 | ✗ |

**设计原则**：
- 优先使用 **public 继承** 表示 IS-A 关系
- 对于 HAS-A 关系，优先使用**组合**而非 private 继承
- protected/private 继承主要用于代码复用，不推荐滥用

### friend

**概念**：friend 声明允许外部函数或类访问私有成员。

**使用场景**：
- 运算符重载
- 工厂模式
- 测试代码
- 两个紧密相关的类

```cpp
// 1. 友元函数
class Complex {
    double real, imag;
public:
    Complex(double r, double i) : real(r), imag(i) {}

    // 声明为友元
    friend Complex operator+(const Complex& a, const Complex& b);
    friend std::ostream& operator<<(std::ostream& os, const Complex& c);
};

Complex operator+(const Complex& a, const Complex& b) {
    // 可以访问私有成员
    return Complex(a.real + b.real, a.imag + b.imag);
}

std::ostream& operator<<(std::ostream& os, const Complex& c) {
    return os << "(" << c.real << ", " << c.imag << ")";
}

// 2. 友元类
class Controller;  // 前向声明

class Model {
    friend class Controller;  // Controller 可以访问 Model 的私有成员
private:
    int data;
};

class Controller {
    Model model;
public:
    void update() {
        model.data = 42;  // 可以访问
    }
};

// 3. 友元成员函数
class Display;
class Storage {
    friend void Display::show(const Storage&);  // 仅特定成员函数是友元
private:
    int value;
};

class Display {
public:
    void show(const Storage& s) {
        std::cout << s.value << std::endl;  // 可以访问
    }
};

// 4. 友元与模板
template <typename T>
class Vector {
    friend bool operator==<T>(const Vector&, const Vector&);  // 需要指定模板参数
private:
    T* data;
    size_t size;
};

template <typename T>
bool operator==(const Vector<T>& a, const Vector<T>& b) {
    // 可以访问 a 和 b 的私有成员
    // ...
}
```

**友元的设计考虑**：
- 友元打破了封装，应谨慎使用
- 比将成员设为 public 好，因为关系是显式的
- 运算符重载是最常见的使用场景

## 1.5 面向对象关键词

### virtual

**概念**：virtual 实现运行时多态（动态绑定）。

**底层原理**：
- 每个有虚函数的类有一个**虚函数表**（vtable）
- 每个对象有一个**虚函数表指针**（vptr）
- 调用虚函数时，通过 vptr 查找 vtable 中的函数地址

```cpp
// 1. 虚函数的基本使用
class Animal {
public:
    virtual void speak() {  // 虚函数
        std::cout << "Animal speaks" << std::endl;
    }

    virtual ~Animal() = default;  // 虚析构函数（重要！）
};

class Dog : public Animal {
public:
    void speak() override {  // override 明确表示覆盖
        std::cout << "Woof!" << std::endl;
    }
};

void makeSpeak(Animal* animal) {
    animal->speak();  // 动态绑定
}

int main() {
    Dog dog;
    makeSpeak(&dog);  // 输出: Woof!

    Animal* a = new Dog();
    a->speak();       // 输出: Woof!
    delete a;         // 如果析构函数不是 virtual，可能不会调用 Dog 的析构函数
}

// 2. override 和 final (C++11)
class Base {
public:
    virtual void foo() {}
    virtual void bar() final {}  // final: 禁止进一步覆盖
};

class Derived : public Base {
public:
    void foo() override {}  // override: 编译器检查是否真的覆盖了
    // void bar() override {}  // 错误！bar 是 final 的
    // void baz() override {}  // 错误！Base 中没有 baz
};

// 3. 纯虚函数和抽象类
class Shape {
public:
    virtual double area() const = 0;  // 纯虚函数
    virtual void draw() const {
        std::cout << "Drawing shape" << std::endl;
    }
    virtual ~Shape() = default;
};

// Shape 是抽象类，不能直接实例化
// Shape s;  // 错误！

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() const override {
        return 3.14159 * radius * radius;
    }
};

// 4. 虚函数的默认参数（陷阱！）
class Base {
public:
    virtual void func(int x = 10) {  // 默认参数
        std::cout << "Base: " << x << std::endl;
    }
};

class Derived : public Base {
public:
    void func(int x = 20) override {  // 不同的默认参数
        std::cout << "Derived: " << x << std::endl;
    }
};

Base* b = new Derived();
b->func();  // 输出: Derived: 10
// 默认参数是静态绑定的！使用的是 Base 的默认参数

// 5. 虚函数表模型示意
/*
   class Base {
       virtual void f1() {}
       virtual void f2() {}
   };

   内存布局：
   +----------------+
   | vptr           |  ---> +---------------+
   +----------------+        | &Base::f1()   |
   | other members  |        +---------------+
   +----------------+        | &Base::f2()   |
                            +---------------+
*/
```

**虚函数的性能考虑**：
- 虚函数调用比普通调用慢（需要间接寻址）
- 阻止编译器内联优化
- 每个对象增加一个指针的大小（通常是 8 字节）
- 在性能关键代码中避免过度使用

### explicit

**概念**：explicit 禁止隐式类型转换和隐式调用构造函数。

**使用场景**：

```cpp
// 1. 禁止单参数构造函数的隐式转换
class String {
public:
    explicit String(const char* str) : data(str) {}
    // explicit String(int size) : size(size) {}  // C++11: 多参数也可以 explicit

private:
    std::string data;
};

String s1("hello");    // ✓ 直接初始化
// String s2 = "hello";  // ✗ 错误！禁止隐式转换
String s3 = String("world");  // ✓ 显式转换

void func(String str) {}
// func("test");  // ✗ 错误！
func(String("test"));  // ✓

// 2. explicit 与 bool 转换运算符
class SmartPointer {
    void* ptr;
public:
    explicit operator bool() const {
        return ptr != nullptr;
    }
};

SmartPointer sp;
if (sp) {  // ✓ 上下文转换（允许）
    // ...
}
// int x = sp;  // ✗ 错误！不是 bool 的隐式转换
int y = static_cast<bool>(sp);  // ✓ 显式转换

// 3. C++11: explicit 用于多参数构造函数
class Point {
public:
    explicit Point(double x = 0, double y = 0) : x_(x), y_(y) {}
private:
    double x_, y_;
};

Point p1(1.0, 2.0);   // ✓
// Point p2 = {1.0, 2.0};  // ✗ C++11 起，explicit 禁止列表初始化
Point p3{1.0, 2.0};   // ✓ 直接列表初始化（不受 explicit 影响）
```

**为什么推荐 explicit**：
- 避免意外的类型转换导致错误
- 使代码意图更明确
- 现代 C++ 最佳实践：单参数构造函数应声明为 explicit

### this

**概念**：this 是一个指向当前对象的指针（在非静态成员函数中可用）。

**特点**：
- 是一个右值（prvalue）
- 类型：`ClassName* const`（不能修改 this 指向）
- 在 const 成员函数中类型为 `const ClassName* const`

```cpp
// 1. this 的基本用法
class MyClass {
    int value;
public:
    void setValue(int value) {
        this->value = value;  // 区分成员变量和参数
    }

    MyClass* getPointer() {
        return this;  // 返回指向当前对象的指针
    }

    // const 成员函数中的 this
    int getValue() const {
        // this 的类型是 const MyClass* const
        return this->value;
    }
};

// 2. 返回 *this 实现链式调用
class Builder {
    std::string name;
    int age = 0;
public:
    Builder& setName(const std::string& n) {
        name = n;
        return *this;  // 返回引用
    }

    Builder& setAge(int a) {
        age = a;
        return *this;
    }

    Builder& print() {
        std::cout << name << ", " << age << std::endl;
        return *this;
    }
};

// 使用链式调用
Builder b;
b.setName("Alice").setAge(25).print();

// 3. this 与 const 重载
class Container {
public:
    // 非 const 版本
    int& operator[](size_t index) {
        return data[index];
    }

    // const 版本
    const int& operator[](size_t index) const {
        return data[index];
    }

private:
    std::vector<int> data;
};

// 4. 委托构造函数 (C++11)
class MyClass {
    int x, y;
public:
    MyClass(int x) : MyClass(x, 0) {}  // 委托给另一个构造函数

    MyClass(int x, int y) : x(x), y(y) {}
    // 注意：构造函数委托后，this 的初始化顺序需要注意
};

// 5. this 捕获 (C++17)
class Widget {
    std::string name;
public:
    auto getLambda() {
        return [*this] {  // C++17: 捕获 *this 的副本
            return name;
        };
    }

    auto getLambdaRef() {
        return [this] {  // 捕获 this 指针（可能悬垂）
            return name;
        };
    }
};
```

## 1.7 SFINAE (Substitution Failure Is Not An Error)

**概念**：SFINAE 是 C++ 模板元编程的核心技术，全称是 "替换失败并非错误"。

**核心思想**：在模板参数推导过程中，如果某种替换导致无效代码，编译器不会报错，而是简单地丢弃这个候选函数/模板，继续尝试其他候选。

### 1.7.1 SFINAE 基本原理

```cpp
// 场景：检查类型是否有 value_type 成员
// 假设我们有两个重载函数模板
template <typename T>
typename T::value_type getValue(const T& t) {
    // 如果 T 有 value_type，这个版本可行
    return t.value;
}

template <typename T>
T getValue(const T& t) {
    // 如果 T 没有 value_type，使用这个版本
    return t;
}

// 测试
struct HasValueType {
    using value_type = int;
    int value = 42;
};

struct NoValueType {
    int data = 100;
};

HasValueType h1;
NoValueType n1;

auto x1 = getValue(h1);  // 调用第一个版本，返回类型是 int
auto x2 = getValue(n1);  // 调用第二个版本，返回类型是 NoValueType

// 工作原理：
// 当编译器尝试 getValue(h1) 时：
// 1. 首先尝试第一个模板：T = HasValueType
//    - T::value_type 存在，替换成功 → 这是一个候选
// 2. 然后尝试第二个模板：T = HasValueType
//    - 替换也成功 → 这也是一个候选
// 3. 两个都成功，选择更特化的版本（第一个）
//
// 当编译器尝试 getValue(n1) 时：
// 1. 首先尝试第一个模板：T = NoValueType
//    - T::value_type 不存在，替换失败
//    - SFINAE 生效：不是错误，只是丢弃这个候选
// 2. 然后尝试第二个模板：T = NoValueType
//    - 替换成功 → 使用这个版本
```

### 1.7.2 SFINAE 的触发条件

SFINAE 只在**直接上下文**中发生的替换失败时生效，其他错误仍然是编译错误：

```cpp
template <typename T>
void func() {
    typename T::value_type x;  // 如果 T 没有 value_type，SFINAE 生效

    T::nonexistent_type y;     // ✗ 编译错误！不在直接上下文中

    typedef typename T::type type;  // SFINAE 生效
    type* p;                        // 如果上面失败，这里也会失败
}

// 正确做法：所有检查都要在函数签名中
template <typename T>
auto func() -> typename T::value_type {  // SFINAE 生效
    return T::value_type{};
}
```

### 1.7.3 std::enable_if (C++11)

`std::enable_if` 是实现 SFINAE 的标准工具：

```cpp
// std::enable_if 的定义（简化）
template <bool Condition, typename T = void>
struct enable_if {
    // 当 Condition = false 时，没有 type 成员
};

template <typename T>
struct enable_if<true, T> {
    using type = T;  // 当 Condition = true 时，定义 type
};

// 使用示例 1：启用/禁用函数模板
template <typename T>
typename std::enable_if<std::is_integral<T>::value, T>::type
doubleValue(T t) {
    return t * 2;
}

// 只对整数类型有效
doubleValue(42);       // ✓
doubleValue(3.14);     // ✗ 没有匹配的函数（enable_if 失败，没有 type 成员）

// 使用示例 2：多个重载
template <typename T>
typename std::enable_if<std::is_integral<T>::value>::type
process(T t) {
    std::cout << "Processing integer: " << t << std::endl;
}

template <typename T>
typename std::enable_if<std::is_floating_point<T>::value>::type
process(T t) {
    std::cout << "Processing float: " << t << std::endl;
}

process(42);      // 输出: Processing integer: 42
process(3.14);   // 输出: Processing float: 3.14

// 使用示例 3：默认模板参数（更简洁的语法）
template <typename T,
          typename std::enable_if<std::is_integral<T>::value, int>::type = 0>
void onlyIntegers(T t) {
    std::cout << "Integer: " << t << std::endl;
}

// 使用示例 4：使用 decltype 配合 std::enable_if
template <typename T>
auto onlyIntegers2(T t) -> decltype(std::enable_if<std::is_integral<T>::value>{}, void()) {
    std::cout << "Integer: " << t << std::endl;
}
```

### 1.7.4 现代 SFINAE 技巧

#### 1. 使用 decltype + comma operator

```cpp
// 检查是否有 begin() 方法
template <typename T>
auto has_begin_impl(int) -> decltype(std::begin(std::declval<T>()), std::true_type{});

template <typename T>
auto has_begin_impl(...) -> std::false_type;

template <typename T>
struct has_begin : decltype(has_begin_impl<T>(0)) {};

// 使用
static_assert(has_begin<std::vector<int>>::value, "vector has begin");
static_assert(!has_begin<int>::value, "int has no begin");
```

#### 2. 使用 expression SFINAE (C++11/14)

```cpp
// 检查类型是否有 size() 方法
template <typename T>
class has_size {
    // 尝试替换：如果能调用 t.size()，则返回 sizeof(yes)
    template <typename U>
    static auto check(int) ->
        decltype(std::declval<U>().size(), std::true_type{});

    // 失败时返回 no
    template <typename>
    static std::false_type check(...);

public:
    static constexpr bool value = decltype(check<T>(0))::value;
};

// 使用
static_assert(has_size<std::vector<int>>::value, "vector has size");
```

#### 3. void_t 技巧 (C++17)

```cpp
// std::void_t 的简化实现
template <typename... Ts>
struct make_void {
    using type = void;
};

template <typename... Ts>
using void_t = typename make_void<Ts...>::type;

// 使用 void_t 检测成员存在性
template <typename T, typename = void>
struct has_value_type : std::false_type {};

template <typename T>
struct has_value_type<T, void_t<typename T::value_type>> : std::true_type {};

// 或更简洁的 C++17 变量模板
template <typename T, typename = void>
constexpr bool has_value_type_v = false;

template <typename T>
constexpr bool has_value_type_v<T, void_t<typename T::value_type>> = true;
```

### 1.7.5 C++20 Concepts（SFINAE 的现代替代）

C++20 的 concepts 提供了更清晰的语法：

```cpp
// 定义 concept
template <typename T>
concept Integral = std::is_integral_v<T>;

template <typename T>
concept HasBegin = requires(T t) {
    std::begin(t);
};

// 使用 concept（比 SFINAE 更易读）
template <Integral T>
T doubleValue(T t) {
    return t * 2;
}

// requires 子句
template <typename T>
requires HasBegin<T>
void processContainer(T& container) {
    for (auto& item : container) {
        // ...
    }
}

// concept 比 SFINAE 的优势：
// 1. 语法更清晰
// 2. 错误信息更友好
// 3. 可以组合多个条件
template <typename T>
concept Sortable = std::regular<T> && requires(T a, T b) {
    { a < b } -> std::convertible_to<bool>;
};
```

### 1.7.6 常用 SFINAE 应用场景

```cpp
// 1. 根据类型选择不同实现
template <typename T>
std::string toString(T value) {
    if constexpr (std::is_integral_v<T>) {
        return std::to_string(value);
    } else if constexpr (std::is_same_v<T, std::string>) {
        return value;
    } else {
        return std::string(value);
    }
}

// 2. 条件启用成员函数
template <typename T>
class Container {
    std::vector<T> data;

public:
    void push_back(const T& value) {
        data.push_back(value);
    }

    // 只对可复制类型启用
    template <typename U = T,
              typename = std::enable_if_t<std::is_copy_assignable<U>::value>>
    void insert(const T& value) {
        data.insert(data.begin(), value);
    }
};

// 3. 检测 iterator 类型
template <typename T>
using iterator_type = typename T::iterator;

template <typename T>
using element_type = typename T::value_type;

// 4. SFINAE 友好的 size() 实现
template <typename T>
constexpr auto size(const T& c) -> decltype(c.size()) {
    return c.size();
}

template <typename T, std::size_t N>
constexpr std::size_t size(const T (&)[N]) noexcept {
    return N;
}

// 5. 标签分发 (Tag Dispatching) - 另一种替代 SFINAE 的模式
template <typename T>
void process_impl(T t, std::true_type) {
    std::cout << "Has iterator\n";
}

template <typename T>
void process_impl(T t, std::false_type) {
    std::cout << "No iterator\n";
}

template <typename T>
void process(T t) {
    process_impl(t, std::is_integral<T>{});
}
```

### 1.7.7 SFINAE 总结对比

| 技术 | C++版本 | 优点 | 缺点 |
|------|---------|------|------|
| std::enable_if | C++11 | 标准库支持 | 语法冗长 |
| decltype + SFINAE | C++11 | 灵活 | 难以阅读 |
| void_t | C++17 | 简洁 | 需要自定义（C++17前） |
| Concepts | C++20 | 最清晰、最好错误信息 | 需要 C++20 |
| if constexpr | C++17 | 内联、直观 | 仅函数内 |
| 标签分发 | C++11 | 清晰的控制流 | 需要额外类型特征 |

**SFINAE 核心要点**：
1. **替换失败不是错误**，只是丢弃该候选
2. 只在**直接上下文**中生效
3. 是 C++ 模板元编程的**核心技术**
4. C++20 concepts 提供了**更好的替代方案**
5. 合理使用可以极大提升模板代码的**表达力**

## 1.8 C++11 新增关键词

### constexpr

**概念**：constexpr 声明可以在编译期求值的常量表达式。

**特点**：
- 编译期求值（如果参数是常量）
- 隐式 inline
- 可以用于需要常量表达式的上下文（数组大小、模板参数等）

```cpp
// 1. constexpr 变量
constexpr int max_size = 100;
int arr[max_size];  // ✓ 可以作为数组大小

// 2. constexpr 函数
constexpr int square(int n) {
    return n * n;
}

constexpr int s = square(10);  // 编译期求值
int x = 5;
int y = square(x);             // 运行期求值（x 不是常量）

// 3. constexpr 函数的限制（C++11）
constexpr int factorial(int n) {
    // return n > 0 ? n * factorial(n - 1) : 1;  // C++11: 只能一个 return
    // C++14 起可以有更多语句
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

constexpr int f5 = factorial(5);  // 编译期计算

// 4. constexpr 构造函数
class Point {
    double x_, y_;
public:
    constexpr Point(double x, double y) : x_(x), y_(y) {}
    constexpr double x() const { return x_; }
    constexpr double y() const { return y_; }
};

constexpr Point p{3.0, 4.0};
constexpr double dist = std::sqrt(p.x() * p.x() + p.y() * p.y());

// 5. constexpr if (C++17)
template <typename T>
auto get_value(T t) {
    if constexpr (std::is_pointer_v<T>) {
        return *t;  // 如果 T 是指针类型
    } else {
        return t;   // 否则
    }
}

// 6. constexpr lambda (C++17)
constexpr auto add = [](int a, int b) {
    return a + b;
};
constexpr int sum = add(1, 2);

// 7. constexpr 虚函数 (C++20)
class Base {
public:
    virtual constexpr void func() {}
};

// 8. constexpr 在标准库中的使用
std::array<int, 5> arr{};  // std::array 大小必须是常量表达式
```

**constexpr vs const**：

| 特性 | constexpr | const |
|------|-----------|-------|
| 编译期求值 | 尽可能 | 否 |
| 初始值 | 必须是常量表达式 | 任何表达式 |
| 可修改性 | const | const |
| 用途 | 编译期计算、模板参数 | 运行时常量 |

### decltype

**概念**：decltype 推导表达式的类型。

**推导规则**：

```cpp
// 1. 基本用法
int x = 42;
decltype(x) y = 10;  // y 的类型是 int

const int& r = x;
decltype(r) z = x;   // z 的类型是 const int&

// 2. decltype 与 auto 的区别
int a = 0;
decltype(a) b = a;     // b 是 int
auto c = a;            // c 是 int

int& ref = a;
decltype(ref) d = a;   // d 是 int&（引用）
auto e = ref;          // e 是 int（引用被剥离）

// 3. decltype((var)) 的特殊性
int n = 10;
decltype(n)   n1 = n;  // n1 是 int
decltype((n)) n2 = n;  // n2 是 int&（额外的括号）

// 4. 尾置返回类型 (C++11)
template <typename T, typename U>
auto add(T t, U u) -> decltype(t + u) {
    return t + u;
}

// 5. decltype(auto) (C++14)
auto getValue() {
    int x = 42;
    return (x);  // decltype(auto) 推导为 int&
}

// 正确用法
std::vector<std::string> vec;
decltype(auto) first = vec[0];  // first 是 string&

// 6. 完美转发返回值
template <typename F, typename... Args>
decltype(auto) invoke(F&& f, Args&&... args) {
    return std::forward<F>(f)(std::forward<Args>(args)...);
}

// 7. SFINAE 中的应用
template <typename T>
auto begin(T& t) -> decltype(t.begin()) {
    return t.begin();
}
```

### noexcept

**概念**：noexcept 指定函数不抛出异常。

**用途**：
- 性能优化（编译器可以生成更高效的代码）
- 函数契约（明确表示不会抛出异常）
- 标准库要求（某些操作必须 noexcept）

```cpp
// 1. noexcept 说明符
void func1() noexcept {
    // 承诺不抛出异常
    // 如果抛出，直接调用 std::terminate()
}

void func2() noexcept(true);   // 等价于上面的
void func3() noexcept(false);  // 等价于没有 noexcept

// 2. 条件 noexcept
template <typename T>
void swap(T& a, T& b) noexcept(noexcept(a.swap(b))) {
    a.swap(b);
}
// 只有当 T::swap 不抛出异常时，此 swap 才不抛出异常

// 3. noexcept 运算符
void mightThrow() {}
void wontThrow() noexcept {}

int main() {
    std::cout << noexcept(mightThrow()) << std::endl;  // 输出 0 (false)
    std::cout << noexcept(wontThrow()) << std::endl;   // 输出 1 (true)
}

// 4. noexcept 与移动构造函数
// 标准库容器在重新分配时，如果移动构造函数是 noexcept 的，
// 会优先移动而非复制
class MyClass {
    std::vector<int> data;
public:
    MyClass() = default;

    MyClass(MyClass&& other) noexcept : data(std::move(other.data)) {}

    MyClass& operator=(MyClass&& other) noexcept {
        data = std::move(other.data);
        return *this;
    }
};

// 5. noexcept 在异常规范中的使用
// C++17: noexcept 成为类型系统的一部分
using NoThrowFunc = void(*)() noexcept;
using MayThrowFunc = void(*)();

// NoThrowFunc 和 MayThrowFunc 是不同的类型

// 6. noexcept 与析构函数
// C++11 后，析构函数默认 noexcept
class Widget {
public:
    ~Widget() = default;  // 隐式 noexcept

    // 显式指定可能抛出异常的析构函数（不推荐）
    ~BadWidget() noexcept(false) {
        throw std::runtime_error("destructor throwing");
    }
};
```

**noexcept 的性能影响**：
- 编译器可以省略异常处理的栈展开代码
- 允许更好的优化（如直接终止而非栈展开）
- 对于移动语义很重要（标准库检查 noexcept）

### nullptr

**概念**：nullptr 是空指针字面量，类型为 `std::nullptr_t`。

**相比 NULL/0 的优势**：
- 有明确类型，避免重载歧义
- 不与整数 0 隐式转换

```cpp
// 1. nullptr 的基本使用
int* p1 = nullptr;
std::shared_ptr<int> p2 = nullptr;

// 2. nullptr vs NULL/0
void f(int x);
void f(char* p);

f(0);       // 调用 f(int)，可能不是预期
f(NULL);    // 可能是 int 或 long，取决于实现
f(nullptr); // 明确调用 f(char*)

// 3. nullptr 的类型
std::nullptr_t np = nullptr;
// nullptr_t 可以转换为任何指针类型

int* pi = nullptr;
char* pc = nullptr;
void (*pf)() = nullptr;

// 4. 模板中使用 nullptr
template <typename T>
void setPointer(T* ptr = nullptr) {
    if (ptr == nullptr) {
        // ...
    }
}

// 5. nullptr 与智能指针
std::unique_ptr<int> up = nullptr;  // ✓
// std::unique_ptr<int> up = NULL;  // ✗ 可能编译失败

// 6. 条件判断
int* p = nullptr;
if (p) {
    // p 不是 nullptr
}

if (p != nullptr) {
    // 等价的写法
}

// 7. nullptr 与函数重载
void func(int);
void func(int*);
void func(std::nullptr_t);

func(nullptr);  // 调用 func(std::nullptr_t)
func(0);        // 调用 func(int)
```

### auto

**概念**：auto 让编译器自动推导变量类型。

**C++11 vs C++14/17**：

```cpp
// 1. 基本用法
auto i = 42;                  // int
auto d = 3.14;                // double
auto str = "hello";           // const char*
auto& ref = i;                // int&
auto* ptr = &i;               // int*

// 2. auto 与引用
int x = 10;
auto& r = x;       // r 是 int&
auto c = r;        // c 是 int（引用被剥离）
auto&& rr = x;     // 万能引用，推导为 int&
auto&& rr2 = 10;   // 万能引用，推导为 int&&

// 3. auto 与 const
const int ci = 10;
auto a = ci;       // a 是 int（顶层 const 被剥离）
auto& b = ci;      // b 是 const int&（保留 const）

// 4. C++14: decltype(auto)
auto func() {
    int x = 42;
    return x;           // 返回 int
}

decltype(auto) func2() {
    int x = 42;
    return (x);         // 返回 int&（注意括号！）
}

// 5. C++17: 结构化绑定
std::map<int, std::string> m = {{1, "one"}, {2, "two"}};
for (const auto& [key, value] : m) {
    std::cout << key << ": " << value << std::endl;
}

// 6. auto 的陷阱
std::vector<bool> v = {true, false};
auto x = v[0];  // x 不是 bool&，而是 std::vector<bool>::reference（代理类型）
// 使用 auto& 或 decltype(auto) 可以避免

// 7. auto 在函数返回中的使用
auto add(int a, int b) -> int {  // C++11: 尾置返回类型
    return a + b;
}

auto subtract(int a, int b) {     // C++14: 推导返回类型
    return a - b;
}

// 8. auto 不能用于的情况
// auto x;  // 错误：没有初始化器
struct S {
    static auto i = 0;  // ✓ 静态成员
    auto j = 0;         // C++11 错误，C++14 允许非静态成员 auto
};

// 9. auto 与函数指针
void (*pf)();
auto f2 = pf;  // f2 是 void(*)()

// 10. auto 与模板
template <typename T>
void process(const T& container) {
    for (auto it = container.begin(); it != container.end(); ++it) {
        // it 的类型由编译器推导
    }
}
```

**auto 的使用建议**：
- ✓ 用于明显的类型（如迭代器、范围 for）
- ✓ 用于复杂的模板类型
- ⚠ 避免在需要明确类型的场景使用
- ✗ 不要用 auto 代替明确的设计意图（如 bool、size_t）

### static_assert

**概念**：static_assert 在编译期进行断言检查。

**用法**：

```cpp
// 1. 基本用法
static_assert(sizeof(int) == 4, "int must be 4 bytes");
static_assert(sizeof(void*) == 8, "Must be 64-bit");

// 2. 无消息的 static_assert (C++17)
static_assert(sizeof(long) >= 8);  // 消息可选

// 3. 模板中的 static_assert
template <typename T>
void process() {
    static_assert(std::is_integral_v<T>, "T must be integral type");

template <typename T>
constexpr bool is_integral_v = std::is_integral<T>::value;

void process(T t) {
    // 只对整数类型有效
}

// 5. 编译期条件检查
template <size_t N>
struct Factorial {

    static_assert(std::is_default_constructible_v<T>,

    static_assert(std::is_copy_assignable_v<T>,
                  "T must be copy assignable");
};
```

## 1.9 C++20 新增关键词

### concept

**概念**：concept 定义模板参数的约束，使模板更易读和更好的错误信息。

```cpp
// 1. 定义 concept
template <typename T>
concept Integral = std::is_integral_v<T>;

template <typename T>
concept Addable = requires(T a, T b) {
    a + b;  // 必须支持 +
};

// 2. 使用 concept
template <Integral T>
T add(T a, T b) {
    return a + b;
}

// 或使用简写语法
template <typename T>
requires Integral<T>
T add(T a, T b) {
    return a + b;
}

// 更简洁的语法
T add(Integral auto a, Integral auto b) {
    return a + b;
}

// 3. requires 子句
template <typename T>
requires sizeof(T) >= 4
void process(T) {}

// 4. requires 表达式
template <typename T>
concept HasSize = requires(T t) {
    { t.size() } -> std::convertible_to<size_t>;
    { t.empty() } -> std::same_as<bool>;
};

// 5. 组合 concept
template <typename T>
concept Sortable = std::regular<T> && requires(T a, T b) {
    { a < b } -> std::convertible_to<bool>;
};

// 6. concept 与重载
template <std::integral T>
void func(T t) {
    std::cout << "Integral: " << t << std::endl;
}

template <std::floating_point T>
void func(T t) {
    std::cout << "Floating: " << t << std::endl;
}

// 7. 标准 concepts
// std::integral, std::floating_point
// std::regular, std::semiregular
// std::copyable, std::movable
// std::equality_comparable, std::totally_ordered
// 等等...
```

### requires

**概念**：requires 指定模板参数的约束。


### consteval

**概念**：consteval 声明立即函数，必须在编译期求值。

```cpp

// constexpr 可以在编译期或运行期执行

constexpr int maybeCompileTime(int n) {

// 4. is_constant_evaluated()
constexpr bool is_consteval() {
```

### 协程关键词: co_await, co_yield, co_return

**概念**：协程是可挂起和恢复的函数。

```cpp
// 1. 协程的基本结构
Task<int> coroutine() {
    co_return 42;  // co_return: 返回值
}

// 2. co_yield: 挂起并产生值
Generator<int> range(int start, int end) {
    while (start < end) {
        co_yield start++;  // 挂起并返回当前值
    }
}

// 使用
for (auto val : range(1, 10)) {
    std::cout << val << std::endl;
}

// 3. co_await: 等待异步操作
Task<void> asyncFunction() {
    auto result = co_await asyncOperation();  // 挂起，等待完成
    process(result);
}

// 4. 简单的 Task 实现
template <typename T>
struct Task {
    struct promise_type {
        T value_;
        Task get_return_object() {
            return Task{std::coroutine_handle<promise_type>::from_promise(*this)};
        }
        std::suspend_never initial_suspend() { return {}; }
        std::suspend_always final_suspend() noexcept { return {}; }
        void return_value(T value) { value_ = value; }
        void unhandled_exception() { std::terminate(); }
    };

    std::coroutine_handle<promise_type> handle_;

    Task(std::coroutine_handle<promise_type> h) : handle_(h) {}
    ~Task() { if (handle_) handle_.destroy(); }

    T get() {
        if (!handle_.done())
            handle_.resume();
        return handle_.promise().value_;
    }
};
```

## 二、值类别专题 (Value Categories)

### 2.1 历史演进

C++ 的值类别经历了从 C++03 到 C++11 的重大变革：

```
C++03:                    C++11:
┌────────────┐           ┌─────────────────┐
│            │           │    expression   │
│ expression │           │      /   \      │
│   /   \    │           │     /     \     │
│lvalue rvalue│          │ glvalue rvalue  │
└────────────┘           │  /  \    /  \   │
                        │lvalue xvalue xvalue prvalue
                        └─────────────────┘
```

### 2.2 三大基本类别

#### 左值 (lvalue - locator value)

**定义**：表示一个**有身份**的对象或函数。

**特征**：
- 有名称或可通过指针访问
- 有持久的存储（不是临时对象）
- 可以取地址（& 运算符）

```cpp
// lvalue 示例
int x = 42;           // x 是 lvalue
int* p = &x;          // ✓ 可以取地址
x = 10;               // ✓ 可以赋值（如果非 const）

struct Point { int x, y; };
Point pt;             // pt 是 lvalue
pt.x = 5;             // pt.x 是 lvalue

std::string str = "hello";
str[0] = 'H';         // str[0] 返回 lvalue 引用

// 函数名是 lvalue
void foo() {}
void (*pf)() = foo;   // foo 是 lvalue

// 字符串字面量是 lvalue（特殊情况）
"hello";              // const char[6] 类型，是 lvalue
```

#### 纯右值 (prvalue - pure rvalue)

**定义**：用于**初始化**对象或计算其值。

**特征**：
- 没有身份（不能取地址）
- 通常是临时值或字面量
- 用于移动语义的源

```cpp
// prvalue 示例
42;                   // 整数字面量
3.14;                 // 浮点数字面量
true;                 // 布尔字面量

int x = 10;
int y = x + 20;       // x + 20 的结果是 prvalue

std::string("hello"); // 临时对象，prvalue

// 函数返回非引用类型
int add(int a, int b) {
    return a + b;
}
int result = add(1, 2);  // add(1, 2) 是 prvalue

// lambda 表达式
[]() { return 42; };  // prvalue

// this 指针
class MyClass {
    void foo() {
        // this 是 prvalue
    }
};
```

#### 亡值 (xvalue - expiring value)

**定义**：表示资源可以**被重用**的对象（接近生命周期末端）。

**特征**：
- 有身份（可以取地址）
- 可以被移动（资源可重用）
- 通常由 `std::move()` 产生

```cpp
// xvalue 示例
std::string str = "hello";
std::move(str);       // 返回 std::string&&，是 xvalue

// 返回右值引用的函数
std::string&& getString() {
    return std::string("world");
}
getString();          // xvalue

// 访问 xvalue 的成员
struct Point { int x, y; };
Point pt{1, 2};
std::move(pt).x;      // xvalue 的成员，是 xvalue

// 数组的 xvalue
int arr[3] = {1, 2, 3};
std::move(arr);       // int(&&)[3]，xvalue
```

### 2.3 混合类别

#### 泛左值 (glvalue - generalized lvalue)

**定义**：glvalue = lvalue + xvalue

**特征**：有身份的对象

```cpp
// glvalue 示例
int x = 42;           // x 是 lvalue，也是 glvalue
std::string s = "hi";
std::move(s);         // xvalue，也是 glvalue
```

#### 右值 (rvalue)

**定义**：rvalue = prvalue + xvalue

**特征**：可以移动的值

```cpp
// rvalue 示例
42;                   // prvalue，也是 rvalue
std::string("hello"); // prvalue，也是 rvalue
std::move(str);       // xvalue，也是 rvalue
```

### 2.4 值类别矩阵

| 类别 | 有身份 | 可移动 | 可取地址 | 示例 |
|------|--------|--------|----------|------|
| lvalue | ✓ | ✗ | ✓ | 变量名、解引用指针 |
| prvalue | ✗ | ✓ | ✗ | 字面量、临时对象、函数返回值 |
| xvalue | ✓ | ✓ | ✓ | std::move()、返回右值引用 |
| glvalue | ✓ | - | ✓ | lvalue + xvalue |
| rvalue | - | ✓ | - | prvalue + xvalue |

### 2.5 值类别判断规则

```cpp
// 判断表达式值类别的方法
template <typename T>
struct value_category;

// lvalue
template <typename T>
struct value_category<T&> {
    static constexpr const char* value = "lvalue";
};

// prvalue
template <typename T>
struct value_category<T> {
    static constexpr const char* value = "prvalue";
};

// xvalue
template <typename T>
struct value_category<T&&> {
    static constexpr const char* value = "xvalue";
};

// 使用辅助函数
template <typename T>
constexpr const char* getValueCategory(T&&) {
    using Cat = decltype(std::forward<T>(std::declval<T>()));
    return value_category<Cat>::value;
}

// 测试
int x = 10;
getValueCategory(x);       // "lvalue"
getValueCategory(10);      // "prvalue"
getValueCategory(std::move(x));  // "xvalue"
```

### 2.6 decltype 规则

```cpp
// decltype 与值类别
int x = 42;

decltype(x)   y1 = x;     // int（x 是 lvalue）
decltype((x)) y2 = x;     // int&（注意括号！）

// decltype((variable)) 总是引用
// decltype(variable) 是变量的声明类型
```

**decltype 规则总结**：

| 表达式 e | decltype(e) |
|----------|-------------|
| 标识符（如 x） | 变量类型 |
| (x) | T&（左值引用） |
| *p | T&（左值引用） |
| ++x | T&（左值引用） |
| x++ | T（prvalue） |
| std::move(x) | T&&（右值引用） |

### 2.7 标准库工具

```cpp
std::is_rvalue_reference_v<int&&>;     // true

std::remove_reference_t<int&&>;        // int

```

## 三、引用专题

### 3.1 左值引用 (T&)

**定义**：左值引用是对具名对象的别名。

**特性**：
- 必须初始化
- 不能重新绑定
- 不能为空
- 绑定到 lvalue

```cpp
// 1. 基本用法
int x = 10;
int& ref = x;   // ref 是 x 的别名

ref = 20;       // 修改 x
std::cout << x; // 输出 20

// 2. const 引用
const int& cref = x;
// cref = 30;   // 错误！不能通过 const 引用修改

// 3. 引用绑定规则
int& r1 = x;           // ✓
// int& r2 = 10;       // ✗ 不能绑定 prvalue
const int& r3 = 10;    // ✓ const 引用可以绑定 prvalue

// 4. 引用的生命周期延长
const std::string& str = std::string("hello");
// 临时对象生命周期被延长到 str 的作用域结束

int&& rr = 10;         // 右值引用也延长生命周期

// 5. 引用与函数参数
void byValue(int x) {}         // 复制
void byRef(int& x) {}          // 引用（避免复制，可修改）
void byConstRef(const int& x) {}// 引用（避免复制，不可修改）

// 6. 引用与函数返回
int& getMax(int& a, int& b) {
    return (a > b) ? a : b;  // 返回引用
}

// 注意：不要返回局部变量的引用！
// int& badFunc() {
//     int x = 10;
//     return x;  // 危险！返回悬垂引用
// }

// 7. 引用与指针
int* ptr = &x;
int& ref2 = *ptr;  // 从指针创建引用

// 8. 引用 vs 指针
// 引用：
// - 必须初始化
// - 不能为空
// - 不能重新绑定
// - 语法更简洁

// 指针：
// - 可以不初始化
// - 可以为空
// - 可以重新指向
// - 需要解引用语法
```

### 3.2 右值引用 (T&&)

**定义**：右值引用绑定到临时对象或可移动的对象。

**用途**：
- 移动语义
- 完美转发

```cpp
// 1. 右值引用的基本用法
int&& rref1 = 10;              // ✓ 绑定 prvalue
int x = 10;
int&& rref2 = std::move(x);    // ✓ 绑定 xvalue

// 2. 移动构造函数
class String {
    char* data;
    size_t size;
public:
    // 移动构造函数
    String(String&& other) noexcept
        : data(other.data), size(other.size) {
        other.data = nullptr;
        other.size = 0;
    }

    // 移动赋值运算符
    String& operator=(String&& other) noexcept {
        if (this != &other) {
            delete[] data;
            data = other.data;
            size = other.size;
            other.data = nullptr;
            other.size = 0;
        }
        return *this;
    }
};

// 3. 右值引用作为函数参数
void process(int&& x) {
    // x 可以被修改（因为是命名变量）
    x = 20;
}

process(10);  // ✓

// 4. std::move
template <typename T>
typename std::remove_reference<T>::type&& move(T&& t) noexcept {
    return static_cast<typename std::remove_reference<T>::type&&>(t);
}

// 使用
String str1 = "hello";
String str2 = std::move(str1);  // 调用移动构造函数
// str1 现在处于有效但未定义的状态
```

### 3.3 引用折叠

**定义**：在模板参数推导和 typedef 中，多层引用会按规则折叠。

**规则**：

| 输入 | 结果 |
|------|------|
| T& & | T& |
| T& && | T& |
| T&& & | T& |
| T&& && | T&& |

**口诀**：只要有左值引用，结果就是左值引用。只有两个右值引用才折叠为右值引用。

```cpp
// 1. typedef 中的引用折叠
typedef int&  lref;
typedef lref&  lref2;   // int&
typedef lref&& lref3;   // int&（int& && 折叠为 int&）

typedef int&& rref;
typedef rref&  rref2;   // int&（int&& & 折叠为 int&）
typedef rref&& rref3;   // int&&（int&& && 折叠为 int&&）

// 2. 模板中的引用折叠
template <typename T>
void foo(T&& t) {
    // 如果 T 是 int&，T&& 就是 int& && → int&
    // 如果 T 是 int，T&& 就是 int&&
}

int x = 10;
foo(x);       // T 推导为 int&，参数类型 int&
foo(10);      // T 推导为 int，参数类型 int&&

// 3. std::forward 的实现依赖于引用折叠
template <typename T>
T&& forward(std::remove_reference_t<T>& t) noexcept {
    return static_cast<T&&>(t);
}

// 当 T 是 int&：
// 返回类型是 int& && → int&（保持左值引用）

// 当 T 是 int：
// 返回类型是 int&&（右值引用）
```

### 3.4 万能引用/转发引用 (Universal/Forwarding Reference)

**定义**：在模板推导上下文中的 `T&&`，可以绑定到任意值类别。

**识别条件**：
1. 必须是 `T&&` 形式
2. `T` 必须是推导类型
3. 没有 const/volatile 修饰

```cpp
// 1. 万能引用
template <typename T>
void wrapper(T&& arg) {  // ✓ 万能引用
    // arg 可以绑定 lvalue 或 rvalue
}

// 2. 不是万能引用的情况
template <typename T>
class MyClass {
    template <typename U>
    void method(U&& arg) { }  // ✓ 万能引用（U 是推导的）

    void other(std::vector<T>&& arg) { }  // ✗ 右值引用（T 已确定）
};

template <typename T>
void wrong(const T&& arg) { }  // ✗ 有 const，不是万能引用

// 3. 万能引用 vs 右值引用
template <typename T>
void universal(T&& t) {  // 万能引用
    std::cout << "Universal: T is deduced\n";
}

void specific(int&& t) {  // 右值引用
    std::cout << "Specific: int&&\n";
}

// 4. auto&& 也是万能引用
auto&& x1 = 10;   // x1 是 int&&
int y = 20;
auto&& x2 = y;    // x2 是 int&（万能引用）

// 5. 范围 for 中的 auto&&
std::vector<std::string> vec = {"a", "b", "c"};
for (auto&& item : vec) {  // item 是万能引用
    // 可以绑定到 vec 的左值引用或移动后的右值引用
}
```

### 3.5 完美转发 (Perfect Forwarding)

**概念**：将参数以原始的值类别（左值/右值）转发给另一个函数。

**实现**：

```cpp
// 1. std::forward 的实现
template <typename T>
constexpr T&& forward(std::remove_reference_t<T>& arg) noexcept {
    return static_cast<T&&>(arg);
}

template <typename T>
constexpr T&& forward(std::remove_reference_t<T>&& arg) noexcept {
    return static_cast<T&&>(arg);
}

// 2. 完美转发示例
void process(int& x) {
    std::cout << "Lvalue: " << x << std::endl;
}

void process(int&& x) {
    std::cout << "Rvalue: " << x << std::endl;
}

template <typename T>
void wrapper(T&& arg) {
    process(std::forward<T>(arg));  // 完美转发
}

// 使用
int x = 10;
wrapper(x);       // 输出 "Lvalue: 10"
wrapper(20);      // 输出 "Rvalue: 20"

// 3. 可变参数模板的完美转发
template <typename... Args>
void forwardToFunc(Args&&... args) {
    targetFunc(std::forward<Args>(args)...);
}

// 4. 实际应用：make_unique
template <typename T, typename... Args>
std::unique_ptr<T> make_unique(Args&&... args) {
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
}

// 5. 完美转发的陷阱
template <typename T>
void bad(T&& arg) {
    someFunc(arg);  // ✗ arg 总是 lvalue
}

template <typename T>
void good(T&& arg) {
    someFunc(std::forward<T>(arg));  // ✓ 保持原始值类别
}

// 6. 引用折叠在转发中的作用
void foo(int&);
void foo(const int&);

template <typename T>
void bar(T&& arg) {
    foo(std::forward<T>(arg));
}

int x = 10;
bar(x);        // T 推导为 int&，调用 foo(int&)
bar(20);       // T 推导为 int，调用 foo(const int&)
```

### 3.6 返回值优化 (RVO/NRVO)

**RVO (Return Value Optimization)**：编译器优化，省略拷贝/移动。

**NRVO (Named Return Value Optimization)**：命名返回值优化。

```cpp
// 1. RVO
std::string createString() {
    return std::string("hello");  // prvalue
}
// C++17 起，prvalue 的拷贝/移动被强制省略

// 2. NRVO
std::string createNamedString() {
    std::string result = "hello";
    return result;  // NRVO（编译器可选）
}

// 3. 移动语义与 RVO
class Widget {
public:
    Widget() = default;
    Widget(const Widget&) { std::cout << "Copy\n"; }
    Widget(Widget&&) noexcept { std::cout << "Move\n"; }
};

Widget makeWidget() {
    Widget w;
    return w;  // NRVO 可能省略移动
}

// 4. 不满足 NRVO 条件的情况
Widget makeWidgetConditional(bool flag) {
    Widget w1, w2;
    if (flag)
        return w1;  // 可能无法 NRVO
    else
        return w2;
}

// 5. std::move 阻止 RVO/NRVO
Widget badWidget() {
    Widget w;
    return std::move(w);  // ✗ 阻止 RVO/NRVO，不必要的移动
}

// 6. C++17 保证的拷贝省略
struct A {
    A(int) {}
    A(const A&) = delete;
    A(A&&) = delete;
};

A makeA() {
    return A(42);  // ✓ C++17 保证不调用被删除的构造函数
}
```

### 3.7 悬垂引用

**定义**：引用已销毁的对象。

```cpp
// 1. 返回局部变量的引用
int& badFunc() {
    int x = 10;
    return x;  // ✗ 悬垂引用
}

// 2. 引用临时对象
const int& getRef() {
    return 10;  // ✓ 临时对象生命周期延长
}
// 但对于函数返回的临时对象：
const std::string& getString() {
    return std::string("hello");  // ✗ 悬垂引用！
    // 临时对象在表达式结束时销毁
}

// 3. 正确做法
std::string getString() {
    return std::string("hello");  // ✓ 返回值
}

const std::string& getStringRef(const std::string& str) {
    return str;  // ✓ 引用参数
}

// 4. 容器中的悬垂引用
std::vector<int*> getPointers() {
    std::vector<int*> result;
    {
        int x = 10;
        result.push_back(&x);  // ✗ 悬垂指针
    }
    return result;
}

// 5. 智能指针避免悬垂
std::vector<std::shared_ptr<int>> getShared() {
    std::vector<std::shared_ptr<int>> result;
    {
        auto p = std::make_shared<int>(10);
        result.push_back(p);  // ✓ 引用计数增加
    }
    return result;
}
```

### 3.8 引用总结对比

| 特性 | 左值引用 T& | 右值引用 T&& | const T& |
|------|-------------|--------------|----------|
| 绑定到 lvalue | ✓ | ✓ | ✓ |
| 绑定到 prvalue | ✗ | ✓ | ✓ |
| 绑定到 xvalue | ✗ | ✓ | ✓ |
| 可修改对象 | ✓ | ✓ | ✗ |
| 延长临时对象生命周期 | ✗ | ✓ | ✓ |
| 主要用途 | 避免拷贝、修改参数 | 移动语义 | 避免拷贝、只读参数 |

## 四、RAII 与智能指针

### 4.1 RAII 原则

**RAII (Resource Acquisition Is Initialization)**：资源获取即初始化。

**核心思想**：将资源的生命周期与对象的生命周期绑定。

```cpp
// 1. 手动管理资源的问题
void badExample() {
    int* arr = new int[100];
    // 如果发生异常，arr 不会被释放
    // ...
    delete[] arr;
}

void goodExample() {
    std::vector<int> arr(100);  // 自动管理内存
    // 异常安全
}

// 2. RAII 实现
class FileHandle {
    FILE* file;
public:
    explicit FileHandle(const char* filename) {
        file = fopen(filename, "r");
        if (!file) throw std::runtime_error("Failed to open file");
    }

    ~FileHandle() {
        if (file) fclose(file);  // 析构时自动关闭
    }

    // 禁止拷贝
    FileHandle(const FileHandle&) = delete;
    FileHandle& operator=(const FileHandle&) = delete;

    // 允许移动
    FileHandle(FileHandle&& other) noexcept : file(other.file) {
        other.file = nullptr;
    }
};

// 3. 标准库中的 RAII
std::vector<int> v;        // 内存管理
std::mutex mtx;            // lock_guard
std::fstream file;         // 文件管理
std::lock_guard lock(mtx); // 自动解锁
```

### 4.2 unique_ptr

**特点**：
- 独占所有权
- 不可拷贝，可移动
- 无额外开销

```cpp
// 1. 基本用法
std::unique_ptr<int> up1(new int(42));
std::unique_ptr<int> up2 = std::make_unique<int>(42);  // C++14

// 2. 移动语义
std::unique_ptr<int> up3 = std::move(up1);  // ✓
// std::unique_ptr<int> up4 = up1;          // ✗ 不可拷贝

// 3. 自定义删除器
std::unique_ptr<FILE, decltype(&fclose)> file(fopen("test.txt", "w"), fclose);

// 或者用 lambda
auto deleter = [](FILE* f) { fclose(f); };
std::unique_ptr<FILE, decltype(deleter)> file2(fopen("test.txt", "w"), deleter);

// 4. 数组支持
std::unique_ptr<int[]> arr(new int[100]);
arr[0] = 10;

// 5. get() 和 reset()
int* raw = up1.get();      // 获取原始指针
up1.reset();               // 释放并置空
up1.reset(new int(50));    // 释放并重新设置

// 6. release() - 释放所有权
int* rawPtr = up1.release();  // up1 不再拥有指针
delete rawPtr;

// 7. unique_ptr 与工厂模式
class Base {
public:
    virtual ~Base() = default;
    virtual void foo() = 0;
};

class Derived : public Base {
public:
    void foo() override { std::cout << "Derived\n"; }
};

std::unique_ptr<Base> createDerived() {
    return std::make_unique<Derived>();  // ✓ 可以返回
}

// 8. unique_ptr 的实现原理（简化）
template <typename T>
class UniquePtr {
    T* ptr;
public:
    explicit UniquePtr(T* p = nullptr) : ptr(p) {}
    ~UniquePtr() { delete ptr; }

    // 禁止拷贝
    UniquePtr(const UniquePtr&) = delete;
    UniquePtr& operator=(const UniquePtr&) = delete;

    // 移动构造
    UniquePtr(UniquePtr&& other) noexcept : ptr(other.ptr) {
        other.ptr = nullptr;
    }

    // 移动赋值
    UniquePtr& operator=(UniquePtr&& other) noexcept {
        if (this != &other) {
            delete ptr;
            ptr = other.ptr;
            other.ptr = nullptr;
        }
        return *this;
    }

    T& operator*() const { return *ptr; }
    T* operator->() const { return ptr; }
};
```

### 4.3 shared_ptr

**特点**：
- 共享所有权
- 引用计数
- 线程安全的引用计数（非线程安全的对象访问）

```cpp
// 1. 基本用法
std::shared_ptr<int> sp1 = std::make_shared<int>(42);
std::shared_ptr<int> sp2 = sp1;  // 引用计数变为 2

std::cout << sp1.use_count();  // 输出引用计数

// 2. 构造方式
std::shared_ptr<int> p1(new int(10));      // 两次分配
std::shared_ptr<int> p2 = std::make_shared<int>(10);  // 一次分配（推荐）

// 3. 自定义删除器
std::shared_ptr<FILE> file(fopen("test.txt", "w"), [](FILE* f) {
    std::cout << "Closing file\n";
    fclose(f);
});

// 4. 别名构造（aliasing constructor）
struct Owner { int value; };
struct Observer { void observe() {} };

std::shared_ptr<Owner> owner = std::make_shared<Owner>();
std::shared_ptr<int> observer(owner, &owner->value);
// observer 拥有 owner，但指向 owner->value

// 5. shared_ptr 的线程安全
// 引用计数的增减是原子操作（线程安全）
// 对象的访问不是线程安全的
std::shared_ptr<int> sp = std::make_shared<int>(42);

// 线程1
auto local1 = sp;  // ✓ 线程安全
*sp = 10;          // ✗ 需要同步

// 6. weak_ptr 解决循环引用
class Node {
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev;  // 使用 weak_ptr
public:
    Node() { std::cout << "Node created\n"; }
    ~Node() { std::cout << "Node destroyed\n"; }
};

// 7. shared_ptr 的实现原理（简化）
template <typename T>
class SharedPtr {
    T* ptr;
    ControlBlock* ctrl;  // 控制块：引用计数、弱计数、删除器
public:
    explicit SharedPtr(T* p) : ptr(p), ctrl(new ControlBlock{1, 0}) {}

    ~SharedPtr() {
        if (ctrl->refCount.fetch_sub(1) == 1) {
            delete ptr;
            if (ctrl->weakCount == 0) {
                delete ctrl;
            }
        }
    }

    SharedPtr(const SharedPtr& other) : ptr(other.ptr), ctrl(other.ctrl) {
        ctrl->refCount.fetch_add(1);
    }
};
```

```cpp
// 6. 线程池（简化版）
class ThreadPool {
    std::vector<std::thread> workers;
    std::queue<std::function<void()>> tasks;
    std::mutex queue_mutex;
    std::condition_variable cv;
    bool stop = false;
public:
    ThreadPool(size_t threads) {
        for (size_t i = 0; i < threads; ++i) {
            workers.emplace_back([this] {
                while (true) {
                    std::function<void()> task;
                    {
                        std::unique_lock lock(queue_mutex);
                        cv.wait(lock, [this] {
                            return stop || !tasks.empty();
                        });
                        if (stop && tasks.empty()) return;
                        task = std::move(tasks.front());
                        tasks.pop();
                    }
                    task();
                }
            });
        }
    }

    template <typename F>
    void enqueue(F&& f) {
        {
            std::unique_lock lock(queue_mutex);
            tasks.emplace(std::forward<F>(f));
        }
        cv.notify_one();
    }

    ~ThreadPool() {
        {
            std::unique_lock lock(queue_mutex);
            stop = true;
        }
        cv.notify_all();
        for (auto& worker : workers) {
            worker.join();
        }
    }
};
```

```cpp
            queue.push(std::move(value));

        cv.notify_one();
```

# 第二部分：设计模式

> 设计模式是软件设计中常见问题的典型解决方案。23种 GoF 设计模式分为创建型、结构型和行为型三类。本部分详细介绍每个模式的主要用途、与其他模式的比较以及联合使用方式。

---

## 一、创建型模式

创建型模式关注对象的创建过程，将对象的创建和使用分离，降低系统的耦合度。

### 1.1 单例模式 (Singleton)

#### 主要用途和作用

**核心目的**：确保一个类在整个系统中只有一个实例存在，并提供一个全局访问点。

**适用场景**：
- 配置管理器 - 系统只需要一份配置
- 日志系统 - 避免日志文件竞争
- 数据库连接池 - 统一管理连接
- 线程池 - 全局共享的线程资源
- 设备管理器 - 硬件资源唯一访问

**作用机制**：
1. 私有化构造函数，防止外部创建实例
2. 内部维护唯一实例
3. 提供静态公共访问方法
4. 禁止拷贝和赋值

```cpp
// Meyer's Singleton - 现代C++推荐写法
class Singleton {
public:
    static Singleton& getInstance() {
        static Singleton instance;  // C++11保证线程安全的初始化
        return instance;
    }

    void doSomething() {
        std::cout << "Singleton working\n";
    }

private:
    Singleton() = default;                     // 私有构造
    ~Singleton() = default;
    Singleton(const Singleton&) = delete;     // 禁止拷贝
    Singleton& operator=(const Singleton&) = delete;  // 禁止赋值
};

// 使用
Singleton::getInstance().doSomething();
```

#### 与其他模式的比较

| 模式 | 实例数量 | 访问方式 | 使用目的 |
|------|----------|----------|----------|
| **单例** | 1个 | 全局访问点 | 全局唯一资源 |
| **多例(原型)** | 多个(固定) | 管理集合 | 资源池管理 |
| **工厂** | 多个 | 工厂方法创建 | 封装创建逻辑 |
| **享元** | 多个(共享) | 享元工厂 | 减少重复对象 |

#### 与其他模式的联合使用

**与工厂模式联合**：
```cpp
class ConfigManager {
private:
    static ConfigManager* instance;

    // 禁止直接创建，通过工厂获取
    ConfigManager() = default;

    friend class ConfigManagerFactory;
};

class ConfigManagerFactory {
public:
    ConfigManager& createConfig() {
        return ConfigManager::getInstance();
    }
};
```

**与观察者模式联合**：单例作为事件发布中心
```cpp
class EventManager : public Subject {
public:
    static EventManager& getInstance() {
        static EventManager instance;
        return instance;
    }
};
```

#### 实际应用案例
- `std::cout` / `std::cin` - 全局流对象
- Android 的 `ApplicationContext`
- Java Spring 的 `Singleton` Bean
- 游戏引擎中的 `GameManager`

### 1.2 工厂模式 (Factory)

工厂模式包含三种形式：简单工厂、工厂方法、抽象工厂。

#### 1.2.1 简单工厂 (Simple Factory)

**主要用途**：根据参数创建不同类型的产品对象，客户端无需知道具体创建细节。

**适用场景**：
- 产品种类较少且固定
- 创建逻辑简单
- 客户端不关心创建过程

```cpp
enum class ShapeType { CIRCLE, SQUARE, RECTANGLE };

class Shape {
public:
    virtual void draw() = 0;
    virtual ~Shape() = default;
};

class Circle : public Shape {
public:
    void draw() override { std::cout << "Drawing Circle\n"; }
};

class Square : public Shape {
public:
    void draw() override { std::cout << "Drawing Square\n"; }
};

class Rectangle : public Shape {
public:
    void draw() override { std::cout << "Drawing Rectangle\n"; }
};

// 简单工厂
class ShapeFactory {
public:
    static std::unique_ptr<Shape> createShape(ShapeType type) {
        switch (type) {
            case ShapeType::CIRCLE: return std::make_unique<Circle>();
            case ShapeType::SQUARE: return std::make_unique<Square>();
            case ShapeType::RECTANGLE: return std::make_unique<Rectangle>();
            default: return nullptr;
        }
    }
};
```

#### 1.2.2 工厂方法 (Factory Method)

**主要用途**：定义创建对象的接口，让子类决定实例化哪个类。

**适用场景**：
- 产品类型经常增加
- 需要灵活扩展产品族
- 客户端只知道接口

```cpp
// 产品接口
class Product {
public:
    virtual ~Product() = default;
    virtual void use() = 0;
};

// 工厂基类
class Creator {
public:
    virtual ~Creator() = default;
    virtual std::unique_ptr<Product> factoryMethod() = 0;
};

// 具体工厂
class ConcreteCreatorA : public Creator {
public:
    std::unique_ptr<Product> factoryMethod() override {
        return std::make_unique<ConcreteProductA>();
    }
};
```

#### 1.2.3 抽象工厂 (Abstract Factory)

**主要用途**：创建一系列相关或相互依赖对象的接口。

**适用场景**：
- 系统需要多个产品族
- 产品族一起使用
- 需要运行时切换产品族（如跨平台UI）

```cpp
// 抽象工厂
class GUIFactory {
public:
    virtual std::unique_ptr<Button> createButton() = 0;
    virtual std::unique_ptr<Checkbox> createCheckbox() = 0;
};

// Windows 工厂
class WindowsFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<WindowsButton>();
    }
    std::unique_ptr<Checkbox> createCheckbox() override {
        return std::make_unique<WindowsCheckbox>();
    }
};

// Mac 工厂
class MacFactory : public GUIFactory {
    // ... Mac 风格的实现
};
```

#### 工厂模式对比

| 特性 | 简单工厂 | 工厂方法 | 抽象工厂 |
|------|----------|----------|----------|
| **创建方式** | 单一工厂类 | 子类工厂 | 多产品族 |
| **扩展性** | 差(修改工厂) | 好(新增子类) | 好(新增工厂) |
| **复杂度** | 低 | 中 | 高 |
| **产品数量** | 单一产品 | 单一产品 | 产品族 |
| **开闭原则** | 不满足 | 满足 | 满足 |

#### 与其他模式的比较

| 模式 | 相同点 | 不同点 |
|------|--------|--------|
| **工厂 vs 建造者** | 都创建对象 | 工厂一步创建；建造者分步构建复杂对象 |
| **工厂 vs 原型** | 都创建对象 | 工厂直接创建；原型通过复制创建 |
| **工厂 vs 单例** | 都管理对象创建 | 工厂创建多实例；单例只创建一实例 |

#### 与其他模式的联合使用
- **工厂 + 策略模式**：根据配置选择创建策略
- **工厂 + 单例**：工厂创建单例对象
- **工厂 + 注册表**：产品注册表模式

---

### 1.3 建造者模式 (Builder)

#### 主要用途和作用

**核心目的**：将复杂对象的构建与表示分离，使得同样的构建过程可以创建不同的表示。

**适用场景**：
- 创建复杂对象，包含多个组成部分
- 需要支持多种表示方式
- 构建过程需要独立于该对象的组成部分

**作用机制**：
1. **Director** - 指导构建过程
2. **Builder** - 定义构建接口
3. **ConcreteBuilder** - 具体构建实现
4. **Product** - 最终产品

```cpp
// 流式API建造者 (现代C++风格)
class FluentComputerBuilder {
    Computer computer;
public:
    FluentComputerBuilder& setCPU(const std::string& cpu) {
        computer.setCPU(cpu);
        return *this;
    }

    FluentComputerBuilder& setGPU(const std::string& gpu) {
        computer.setGPU(gpu);
        return *this;
    }

    FluentComputerBuilder& setRAM(int ram) {
        computer.setRAM(ram);
        return *this;
    }

    Computer build() {
        return computer;
    }
};

// 使用流式API
Computer pc = FluentComputerBuilder()
    .setCPU("AMD Ryzen 9")
    .setGPU("RX 7900 XTX")
    .setRAM(64)
    .build();
```

#### 与其他模式的比较

| 模式 | 目的 | 构建过程 | 适用对象 |
|------|------|----------|----------|
| **建造者** | 分步构建复杂对象 | 多步骤，可控 | 复杂对象 |
| **工厂** | 封装创建逻辑 | 一步完成 | 任意对象 |
| **原型** | 通过复制创建 | 直接复制 | 相似对象 |

#### 与其他模式的联合使用
- **建造者 + 桥接**：构建复杂的桥接结构
- **建造者 + 组合**：构建树形结构
- **建造者 + 模板方法**：标准化构建流程

---

### 1.4 原型模式 (Prototype)

#### 主要用途和作用

**核心目的**：通过复制现有对象创建新对象，而不是通过 new 创建。

**适用场景**：
- 创建对象成本较高（需要大量初始化）
- 对象类型在运行时确定
- 需要保护原始对象
- 深度复制复杂对象结构

```cpp
// 原型接口
class Prototype {
public:
    virtual std::unique_ptr<Prototype> clone() const = 0;
    virtual ~Prototype() = default;
};

// 具体原型
class ConcretePrototype : public Prototype {
    std::string name;
    std::vector<int> data;

public:
    ConcretePrototype(const std::string& n) : name(n) {
        data = {1, 2, 3, 4, 5};
    }

    // 拷贝构造实现深拷贝
    ConcretePrototype(const ConcretePrototype& other)
        : name(other.name), data(other.data) {}

    std::unique_ptr<Prototype> clone() const override {
        return std::make_unique<ConcretePrototype>(*this);
    }
};
```

#### 与其他模式的比较

| 模式 | 创建方式 | 成本 | 适用场景 |
|------|----------|------|----------|
| **原型** | 复制现有对象 | 低（内存复制） | 相似对象多 |
| **工厂** | 构造新对象 | 高（重新初始化） | 对象差异大 |
| **单例** | 返回唯一实例 | 最低 | 唯一资源 |

#### 与其他模式的联合使用
- **原型 + 工厂**：工厂通过原型创建对象
- **原型 + 组合**：复制复杂的对象树
- **原型 + 备忘录**：实现状态快照

---

## 二、结构型模式

结构型模式关注类和对象的组合，通过继承或组合来构建更大的结构。

### 2.1 适配器模式 (Adapter)

#### 主要用途和作用

**核心目的**：将一个类的接口转换成客户期望的另一个接口，使原本不兼容的类可以一起工作。

**适用场景**：
- 需要使用现有类，但其接口与所需不匹配
- 想要创建可复用的类，与不相关的或不可预见的类（即那些接口可能不一定兼容的类）协同工作
- 需要使用多个现有子类，但通过对每个子类进行子类化来调整它们的接口不现实

**类适配器 vs 对象适配器**：

```cpp
// 目标接口
class Target {
public:
    virtual void request() = 0;
    virtual ~Target() = default;
};

// 被适配者
class Adaptee {
public:
    void specificRequest() {
        std::cout << "Adaptee's specific request\n";
    }
};

// 类适配器（多重继承）
class ClassAdapter : public Target, private Adaptee {
public:
    void request() override {
        specificRequest();  // 调用被适配者的方法
    }
};

// 对象适配器（组合）- 推荐
class ObjectAdapter : public Target {
    Adaptee adaptee;
public:
    void request() override {
        adaptee.specificRequest();
    }
};
```

#### 与其他模式的比较

| 模式 | 目的 | 结构 | 改变接口 |
|------|------|------|----------|
| **适配器** | 接口转换 | 包装 | 是 |
| **装饰器** | 添加职责 | 包装 | 否 |
| **代理** | 控制访问 | 包装 | 否 |
| **桥接** | 分离抽象和实现 | 分离 | 可选 |

#### 与其他模式的联合使用
- **适配器 + 工厂**：创建适配器对象
- **适配器 + 桥接**：适配桥接的一端

---

### 2.2 装饰器模式 (Decorator)

#### 主要用途和作用

**核心目的**：动态地给对象添加额外的职责，比生成子类更灵活。

**适用场景**：
- 需要动态地、透明地给对象添加职责
- 需要动态地撤销添加的职责
- 不能采用生成子类的方法进行扩展（会导致大量子类）

```cpp
// 组件接口
class Component {
public:
    virtual void operation() = 0;
    virtual ~Component() = default;
};

// 具体组件
class ConcreteComponent : public Component {
public:
    void operation() override {
        std::cout << "ConcreteComponent\n";
    }
};

// 装饰器基类
class Decorator : public Component {
    std::unique_ptr<Component> component;
public:
    explicit Decorator(std::unique_ptr<Component> c) : component(std::move(c)) {}

    void operation() override {
        component->operation();
    }
};

// 具体装饰器A
class ConcreteDecoratorA : public Decorator {
public:
    explicit ConcreteDecoratorA(std::unique_ptr<Component> c)
        : Decorator(std::move(c)) {}

    void operation() override {
        addedBehavior();
        Decorator::operation();
    }

private:
    void addedBehavior() {
        std::cout << "DecoratorA added behavior\n";
    }
};

// 具体装饰器B
class ConcreteDecoratorB : public Decorator {
public:
    explicit ConcreteDecoratorB(std::unique_ptr<Component> c)
        : Decorator(std::move(c)) {}

    void operation() override {
        Decorator::operation();
        addedBehavior();
    }

private:
    void addedBehavior() {
        std::cout << "DecoratorB added behavior\n";
    }
};

// 使用 - 多层装饰
auto component = std::make_unique<ConcreteComponent>();
auto decorated = std::make_unique<ConcreteDecoratorB>(
    std::make_unique<ConcreteDecoratorA>(std::move(component))
);
decorated->operation();
// 输出：
// DecoratorA added behavior
// ConcreteComponent
// DecoratorB added behavior
```

#### 与其他模式的比较

| 模式 | 相同点 | 不同点 |
|------|--------|--------|
| **装饰器 vs 适配器** | 都包装对象 | 装饰器添加职责；适配器转换接口 |
| **装饰器 vs 代理** | 都包装对象 | 装饰器添加职责；代理控制访问 |
| **装饰器 vs 组成** | 都可递归 | 装饰器动态；组成静态 |

#### 与其他模式的联合使用
- **装饰器 + 组合**：装饰组合模式中的节点
- **装饰器 + 责任链**：在责任链中添加装饰

---

### 2.3 代理模式 (Proxy)

#### 主要用途和作用

**核心目的**：为其他对象提供一种代理以控制对这个对象的访问。

**适用场景**：
- **远程代理** - 为不同地址空间的对象提供本地代表
- **虚拟代理** - 延迟创建开销大的对象
- **保护代理** - 控制对原始对象的访问权限
- **智能引用** - 替代裸指针，添加额外操作（如引用计数）

```cpp
// 主题接口
class Subject {
public:
    virtual void request() = 0;
    virtual ~Subject() = default;
};

// 真实主题
class RealSubject : public Subject {
public:
    void request() override {
        std::cout << "RealSubject handling request\n";
    }
};

// 代理
class Proxy : public Subject {
    std::unique_ptr<RealSubject> realSubject;
    bool initialized = false;

    void init() {
        if (!initialized) {
            realSubject = std::make_unique<RealSubject>();
            initialized = true;
        }
    }

public:
    void request() override {
        init();  // 延迟初始化
        std::cout << "Proxy: Logging request\n";
        realSubject->request();
        std::cout << "Proxy: Logging response\n";
    }
};

// 使用
Proxy proxy;
proxy.request();  // 第一次会创建 RealSubject
proxy.request();  // 之后直接使用
```

**智能指针作为代理**：
```cpp
// std::unique_ptr, std::shared_ptr 本质上是代理模式
class UniquePtrProxy {
    T* ptr;
public:
    explicit UniquePtrProxy(T* p) : ptr(p) {}
    ~UniquePtrProxy() { delete ptr; }  // 自动管理生命周期

    // 禁止拷贝，允许移动
    UniquePtrProxy(const UniquePtrProxy&) = delete;
    UniquePtrProxy(UniquePtrProxy&& other) noexcept : ptr(other.ptr) {
        other.ptr = nullptr;
    }
};
```

#### 与其他模式的比较

| 模式 | 目的 | 客户端感知 | 控制程度 |
|------|------|------------|----------|
| **代理** | 控制访问 | 不透明 | 高 |
| **装饰器** | 添加职责 | 透明 | 中 |
| **适配器** | 转换接口 | 不透明 | 低 |

#### 与其他模式的联合使用
- **代理 + 观察者**：代理作为被观察者
- **代理 + 装饰器**：代理装饰远程对象

---

### 2.4 桥接模式 (Bridge)

#### 主要用途和作用

**核心目的**：将抽象部分与实现部分分离，使它们可以独立变化。

**适用场景**：
- 不希望在抽象和实现之间有固定的绑定关系
- 类的抽象以及它的实现都应该可以通过子类化独立扩展
- 对一个抽象的实现部分的修改应该对客户不产生影响
- 需要在多个对象间共享实现，但客户不需要知道

```cpp
// 实现接口
class DrawingAPI {
public:
    virtual void drawCircle(double x, double y, double radius) = 0;
    virtual ~DrawingAPI() = default;
};

// 具体实现A
class DirectXAPI : public DrawingAPI {
public:
    void drawCircle(double x, double y, double radius) override {
        std::cout << "DirectX: Circle at (" << x << "," << y
                  << ") r=" << radius << "\n";
    }
};

// 具体实现B
class OpenGLAPI : public DrawingAPI {
public:
    void drawCircle(double x, double y, double radius) override {
        std::cout << "OpenGL: Circle at (" << x << "," << y
                  << ") r=" << radius << "\n";
    }
};

// 抽象类
class Shape {
protected:
    std::shared_ptr<DrawingAPI> drawingAPI;

public:
    explicit Shape(std::shared_ptr<DrawingAPI> api) : drawingAPI(std::move(api)) {}
    virtual void draw() = 0;
    virtual ~Shape() = default;
};

// 具体抽象
class CircleShape : public Shape {
    double x, y, radius;

public:
    CircleShape(double x, double y, double r, std::shared_ptr<DrawingAPI> api)
        : Shape(api), x(x), y(y), radius(r) {}

    void draw() override {
        drawingAPI->drawCircle(x, y, radius);
    }
};

// 使用 - 运行时切换实现
auto dxApi = std::make_shared<DirectXAPI>();
auto glApi = std::make_shared<OpenGLAPI>();

CircleShape circle1(1, 2, 3, dxApi);
circle1.draw();  // DirectX

CircleShape circle2(4, 5, 6, glApi);
circle2.draw();  // OpenGL
```

#### pimpl 惯用法（桥接模式的特例）

```cpp
// 头文件
class MyClass {
public:
    MyClass();
    ~MyClass();
    void doSomething();

private:
    class Impl;
    std::unique_ptr<Impl> pImpl;  // 指向实现的指针
};

// 实现文件
class MyClass::Impl {
    std::string data;
public:
    void doSomething() {
        std::cout << "Impl doing something with " << data << "\n";
    }
};

MyClass::MyClass() : pImpl(std::make_unique<Impl>()) {}
MyClass::~MyClass() = default;

void MyClass::doSomething() {
    pImpl->doSomething();
}
```

#### 与其他模式的比较

| 模式 | 目的 | 分离内容 | 变化方向 |
|------|------|----------|----------|
| **桥接** | 分离抽象和实现 | 抽象/实现 | 两个维度独立变化 |
| **适配器** | 转换接口 | 接口 | 单向转换 |
| **策略** | 封装算法 | 算法 | 算法独立变化 |

#### 与其他模式的联合使用
- **桥接 + 抽象工厂**：工厂创建桥接的实现部分
- **桥接 + 建造者**：建造者构建桥接结构

---

### 2.5 组合模式 (Composite)

#### 主要用途和作用

**核心目的**：将对象组合成树形结构以表示"部分-整体"层次结构，使用户对单个对象和组合对象的使用具有一致性。

**适用场景**：
- 表示对象的部分-整体层次结构
- 希望用户忽略组合对象与单个对象的不同
- 用户统一地使用组合结构中的所有对象

```cpp
// 组件接口
class Graphic {
public:
    virtual void draw() = 0;
    virtual void add(std::shared_ptr<Graphic>) {}
    virtual void remove(std::shared_ptr<Graphic>) {}
    virtual ~Graphic() = default;
};

// 叶子节点
class Circle : public Graphic {
public:
    void draw() override {
        std::cout << "Drawing Circle\n";
    }
};

class Square : public Graphic {
public:
    void draw() override {
        std::cout << "Drawing Square\n";
    }
};

// 组合节点
class CompositeGraphic : public Graphic {
    std::vector<std::shared_ptr<Graphic>> children;

public:
    void add(std::shared_ptr<Graphic> g) override {
        children.push_back(g);
    }

    void remove(std::shared_ptr<Graphic> g) override {
        children.erase(
            std::remove(children.begin(), children.end(), g),
            children.end()
        );
    }

    void draw() override {
        for (const auto& child : children) {
            child->draw();
        }
    }
};

// 使用 - 构建树形结构
auto circle1 = std::make_shared<Circle>();
auto circle2 = std::make_shared<Circle>();
auto square = std::make_shared<Square>();

auto composite = std::make_shared<CompositeGraphic>();
composite->add(circle1);
composite->add(square);

auto composite2 = std::make_shared<CompositeGraphic>();
composite2->add(circle2);
composite2->add(composite);  // 嵌套组合

composite2->draw();
```

#### 与其他模式的比较

| 模式 | 结构 | 客户端使用 | 一致性 |
|------|------|------------|--------|
| **组合** | 树形 | 统一 | 高 |
| **装饰器** | 线性 | 统一 | 中 |
| **链式(责任链)** | 链形 | 不同 | 低 |

#### 与其他模式的联合使用
- **组合 + 迭代器**：遍历组合结构
- **组合 + 访问者**：对组合结构进行操作

---

### 2.6 外观模式 (Facade)

#### 主要用途和作用

**核心目的**：为子系统中的一组接口提供一个统一的高层接口，使子系统更容易使用。

**适用场景**：
- 需要为一个复杂的子系统提供一个简单接口
- 客户程序与抽象类的实现部分之间存在很大的依赖性
- 需要构建一个层次结构的子系统

```cpp
// 复杂的子系统
class CPU {
public:
    void freeze() { std::cout << "CPU frozen\n"; }
    void jump(long position) { std::cout << "CPU jump to " << position << "\n"; }
    void execute() { std::cout << "CPU executing\n"; }
};

class Memory {
public:
    void load(long position, char* data) {
        std::cout << "Memory loading at " << position << "\n";
    }
};

class HardDrive {
public:
    char* read(long lba, int size) {
        std::cout << "HardDrive reading LBA " << lba << "\n";
        return new char[size];
    }
};

// 外观
class ComputerFacade {
    CPU cpu;
    Memory memory;
    HardDrive hardDrive;

public:
    void start() {
        std::cout << "Starting computer...\n";
        cpu.freeze();
        memory.load(0x00, hardDrive.read(0, 1024));
        cpu.jump(0x00);
        cpu.execute();
        std::cout << "Computer started\n";
    }
};

// 使用 - 简化的接口
ComputerFacade computer;
computer.start();
```

#### 与其他模式的比较

| 模式 | 目的 | 接口数量 | 复杂度 |
|------|------|----------|--------|
| **外观** | 简化接口 | 多个统一为一个 | 降低 |
| **适配器** | 转换接口 | 一个转换为另一个 | 不变 |
| **中介者** | 协调交互 | 多个互相引用变为中心引用 | 改变结构 |

#### 与其他模式的联合使用
- **外观 + 单例**：外观作为单例访问子系统
- **外观 + 抽象工厂**：外观使用工厂创建子系统对象

---

### 2.7 享元模式 (Flyweight)

#### 主要用途和作用

**核心目的**：运用共享技术有效地支持大量细粒度的对象。

**适用场景**：
- 一个应用程序使用了大量的对象
- 完全由于使用大量的对象，造成很大的存储开销
- 对象的大多数状态都可变为外部状态
- 如果删除对象的外部状态，那么可以用相对较少的共享对象取代很多组对象

```cpp
// 享元对象
class TreeType {
    std::string name;
    std::string color;
    std::string texture;

public:
    TreeType(std::string n, std::string c, std::string t)
        : name(std::move(n)), color(std::move(c)), texture(std::move(t)) {}

    void draw(int x, int y) const {
        std::cout << "Drawing " << name << " at (" << x << "," << y
                  << ") color=" << color << "\n";
    }
};

// 享元工厂
class TreeFactory {
    std::unordered_map<std::string, std::shared_ptr<TreeType>> treeTypes;

public:
    std::shared_ptr<TreeType> getTreeType(const std::string& name,
                                          const std::string& color,
                                          const std::string& texture) {
        std::string key = name + "-" + color + "-" + texture;
        auto it = treeTypes.find(key);
        if (it != treeTypes.end()) {
            return it->second;
        }
        auto type = std::make_shared<TreeType>(name, color, texture);
        treeTypes[key] = type;
        std::cout << "Creating new TreeType: " << key << "\n";
        return type;
    }
};

// 使用享元的对象
class Tree {
    std::shared_ptr<TreeType> type;
    int x, y;

public:
    Tree(std::shared_ptr<TreeType> t, int x, int y)
        : type(std::move(t)), x(x), y(y) {}

    void draw() const {
        type->draw(x, y);
    }
};

// 使用
TreeFactory factory;
auto treeType1 = factory.getTreeType("Oak", "Green", "Rough");
auto treeType2 = factory.getTreeType("Oak", "Green", "Rough");  // 共享

Tree t1(treeType1, 10, 20);
Tree t2(treeType2, 30, 40);  // 共享相同的 TreeType
```

#### 与其他模式的比较

| 模式 | 目的 | 对象数量 | 状态 |
|------|------|----------|------|
| **享元** | 减少对象数量 | 少量共享 | 内部/外部分离 |
| **单例** | 唯一实例 | 1个 | 全部内部 |
| **原型** | 复制创建 | 多个 | 全部内部 |

#### 与其他模式的联合使用
- **享元 + 组合**：享元作为组合的叶子节点
- **享元 + 工厂**：工厂管理享元对象

## 四、行为型模式

行为型模式关注对象之间的通信和职责分配。

### 4.1 观察者模式 (Observer)

#### 主要用途和作用

**核心目的**：定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并自动更新。

**适用场景**：
- 事件驱动系统（GUI、游戏）
- 消息队列/发布订阅系统
- MVC 架构中的 Model-View 通信
- 分布式系统的状态同步

```cpp
// 观察者接口
class Observer {
public:
    virtual void update(const std::string& message) = 0;
    virtual ~Observer() = default;
};

// 被观察者/主题
class Subject {
    std::vector<std::weak_ptr<Observer>> observers;
    std::mutex mtx;

public:
    void attach(std::shared_ptr<Observer> obs) {
        std::lock_guard lock(mtx);
        observers.push_back(obs);
    }

    void detach(std::shared_ptr<Observer> obs) {
        std::lock_guard lock(mtx);
        observers.erase(
            std::remove_if(observers.begin(), observers.end(),
                [&obs](const std::weak_ptr<Observer>& w) {
                    return w.lock() == obs;
                }),
            observers.end()
        );
    }

    void notify(const std::string& message) {
        std::lock_guard lock(mtx);
        for (auto it = observers.begin(); it != observers.end(); ) {
            if (auto obs = it->lock()) {
                obs->update(message);
                ++it;
            } else {
                it = observers.erase(it);  // 移除已失效的观察者
            }
        }
    }
};

// 具体观察者
class EmailObserver : public Observer {
    std::string email;

public:
    explicit EmailObserver(std::string e) : email(std::move(e)) {}

    void update(const std::string& message) override {
        std::cout << "Email to " << email << ": " << message << "\n";
    }
};
```

#### 与其他模式的比较

| 模式 | 交互模式 | 触发方式 | 依赖方向 |
|------|----------|----------|----------|
| **观察者** | 一对多 | 状态变化 | 观察者→主题 |
| **中介者** | 多对多 | 事件 | 同事→中介者 |
| **发布订阅** | 一对多 | 消息 | 解耦 |

#### 与其他模式的联合使用
- **观察者 + 单例**：主题作为单例
- **观察者 + 命令**：通知时传递命令对象
- **观察者 + 中介者**：中介者使用观察者协调

### 4.2 策略模式 (Strategy)

#### 主要用途和作用

**核心目的**：定义一系列算法，把它们封装起来，并使它们可以互相替换。

**适用场景**：
- 多种算法做同样的事情，只是实现不同
- 需要在运行时动态选择算法
- 算法有复杂的内部结构
- 需要隐藏算法实现细节

```cpp
// 策略接口
class PaymentStrategy {
public:
    virtual void pay(int amount) = 0;
    virtual ~PaymentStrategy() = default;
};

// 具体策略
class CreditCardPayment : public PaymentStrategy {
    std::string cardNumber;

public:
    explicit CreditCardPayment(std::string cn) : cardNumber(std::move(cn)) {}

    void pay(int amount) override {
        std::cout << "Paid " << amount << " using Credit Card " << cardNumber << "\n";
    }
};

class PayPalPayment : public PaymentStrategy {
    std::string email;

public:
    explicit PayPalPayment(std::string e) : email(std::move(e)) {}

    void pay(int amount) override {
        std::cout << "Paid " << amount << " using PayPal " << email << "\n";
    }
};

// 上下文
class ShoppingCart {
    std::unique_ptr<PaymentStrategy> paymentStrategy;

public:
    void setPaymentStrategy(std::unique_ptr<PaymentStrategy> strategy) {
        paymentStrategy = std::move(strategy);
    }

    void checkout(int amount) {
        if (paymentStrategy) {
            paymentStrategy->pay(amount);
        }
    }
};
```

#### 与其他模式的比较

| 模式 | 算法选择 | 切换时机 | 上下文感知 |
|------|----------|----------|------------|
| **策略** | 客户端选择 | 运行时 | 否 |
| **状态** | 状态决定 | 运行时 | 是 |
| **模板方法** | 子类决定 | 编译期 | 是 |

#### 与其他模式的联合使用
- **策略 + 工厂**：工厂创建策略对象
- **策略 + 单例**：策略作为单例
- **策略 + 组合**：多个策略组合

### 4.3 命令模式 (Command)

#### 主要用途和作用

**核心目的**：将请求封装成对象，从而可以用不同的请求对客户进行参数化、对请求排队或记录请求日志，以及支持可撤销的操作。

**适用场景**：
- 需要将调用者和接收者解耦
- 需要在不同的时间指定请求、排队请求、执行请求
- 需要支持撤销(Undo)操作
- 需要支持修改日志

```cpp
// 命令接口
class Command {
public:
    virtual void execute() = 0;
    virtual void undo() = 0;
    virtual ~Command() = default;
};

// 接收者
class Light {
public:
    void on() { std::cout << "Light is ON\n"; }
    void off() { std::cout << "Light is OFF\n"; }
};

// 具体命令
class LightOnCommand : public Command {
    Light& light;

public:
    explicit LightOnCommand(Light& l) : light(l) {}

    void execute() override {
        light.on();
    }

    void undo() override {
        light.off();
    }
};

class LightOffCommand : public Command {
    Light& light;

public:
    explicit LightOffCommand(Light& l) : light(l) {}

    void execute() override {
        light.off();
    }

    void undo() override {
        light.on();
    }
};

// 调用者
class RemoteControl {
    std::unique_ptr<Command> command;
    std::stack<std::unique_ptr<Command>> history;

public:
    void setCommand(std::unique_ptr<Command> cmd) {
        command = std::move(cmd);
    }

    void pressButton() {
        if (command) {
            command->execute();
            history.push(std::move(command));
        }
    }

    void pressUndo() {
        if (!history.empty()) {
            auto top = std::move(const_cast<std::unique_ptr<Command>&>(history.top()));
            history.pop();
            top->undo();
        }
    }
};
```

#### 与其他模式的比较

| 模式 | 目的 | 封装内容 | 撤销支持 |
|------|------|------------|----------|
| **命令** | 请求封装 | 请求 | 容易 |
| **策略** | 算法封装 | 算法 | 困难 |
| **责任链** | 请求处理 | 处理者 | 不支持 |

#### 与其他模式的联合使用
- **命令 + 复合**：宏命令（组合命令）
- **命令 + 备忘录**：实现可撤销操作
- **命令 + 观察者**：通知命令执行

### 4.4 状态模式 (State)

#### 主要用途和作用

**核心目的**：允许一个对象在其内部状态改变时改变它的行为。

**适用场景**：
- 对象的行为依赖于它的状态
- 状态在运行时动态变化
- 操作中有大量与状态相关的条件语句

```cpp
// 状态接口
class State {
public:
    virtual void insertCoin() = 0;
    virtual void ejectCoin() = 0;

    virtual ~State() = default;
};

// 上下文
class VendingMachine {
    std::unique_ptr<State> currentState;

public:
    void setState(std::unique_ptr<State> state) {
        currentState = std::move(state);
    }

    void insertCoin() { currentState->insertCoin(); }
    void ejectCoin() { currentState->ejectCoin(); }
    void pushButton() { currentState->pushButton(); }
};

// 具体状态
class NoCoinState : public State {
    VendingMachine& machine;

public:
    explicit NoCoinState(VendingMachine& m) : machine(m) {}

    void insertCoin() override {
        std::cout << "Coin inserted\n";
        machine.setState(std::make_unique<HasCoinState>(machine));
    }

    void ejectCoin() override {
        std::cout << "No coin to eject\n";
    }

    void pushButton() override {
        std::cout << "Insert coin first\n";
    }
};

class HasCoinState : public State {
    VendingMachine& machine;

public:
    explicit HasCoinState(VendingMachine& m) : machine(m) {}

    void insertCoin() override {
        std::cout << "Coin already inserted\n";
    }

    void ejectCoin() override {
        std::cout << "Coin ejected\n";
        machine.setState(std::make_unique<NoCoinState>(machine));
    }

    void pushButton() override {
        std::cout << "Dispensing product\n";
        machine.setState(std::make_unique<NoCoinState>(machine));
    }
};
```

| **状态** | 当前状态 | 多个 | 内部驱动 |
| **策略** | 客户端 | 多个 | 外部驱动 |

### 4.5 模板方法模式 (Template Method)

#### 主要用途和作用

**核心目的**：定义算法的骨架，将一些步骤延迟到子类。

**适用场景**：
- 算法整体结构固定，部分步骤可变
- 需要在子类中重新定义算法的某些步骤
- 代码复用，避免重复

```cpp
// 抽象类
class DataProcessor {
public:
    void process() {  // 模板方法 - 定义算法骨架
        readData();
        validateData();
        transformData();
        saveData();
    }

protected:
    virtual void readData() = 0;
    virtual void transformData() = 0;
    virtual void saveData() = 0;

    void validateData() {  // 通用步骤，不可覆盖
        std::cout << "Validating data...\n";
    }
};

// 具体实现
class CSVProcessor : public DataProcessor {
protected:
    void readData() override {
        std::cout << "Reading CSV data\n";
    }

    void transformData() override {
        std::cout << "Transforming CSV data\n";
    }

    void saveData() override {
        std::cout << "Saving CSV data\n";
    }
};
```

| 模式 | 算法定义 | 变化部分 | 复用方式 |
|------|------------|----------|----------|
| **模板方法** | 父类 | 子类重写 | 继承 |
| **策略** | 接口 | 策略对象 | 组合 |
| **建造者** | 指导者 | 建造者 | 分离 |

#### 与其他模式的联合使用
- **模板方法 + 工厂方法**：创建对象
- **模板方法 + 策略**：可变步骤使用策略

### 4.6 责任链模式 (Chain of Responsibility)

#### 主要用途和作用

**核心目的**：使多个对象都有机会处理请求，从而避免请求的发送者和接收者耦合。

**适用场景**：
- 有多个对象可以处理请求
- 不知道具体哪个对象处理请求
- 需要动态指定处理者集合

```cpp
// 处理者接口
class Handler {
    std::unique_ptr<Handler> next;

public:
    void setNext(std::unique_ptr<Handler> n) {
        next = std::move(n);
    }

    void handleRequest(int request) {
        if (canHandle(request)) {
            process(request);
        } else if (next) {
            next->handleRequest(request);
        } else {
            std::cout << "No handler found for request: " << request << "\n";
        }
    }

protected:
    virtual bool canHandle(int request) = 0;
    virtual void process(int request) = 0;
};

// 具体处理者
class PositiveHandler : public Handler {
protected:
    bool canHandle(int request) override {
        return request > 0;
    }

    void process(int request) override {
        std::cout << "PositiveHandler processing: " << request << "\n";
    }
};

class NegativeHandler : public Handler {
protected:
    bool canHandle(int request) override {
        return request < 0;
    }

    void process(int request) override {
        std::cout << "NegativeHandler processing: " << request << "\n";
    }
};

class ZeroHandler : public Handler {
protected:
    bool canHandle(int request) override {
        return request == 0;
    }

    void process(int request) override {
        std::cout << "ZeroHandler processing: " << request << "\n";
    }
};
```

#### 与其他模式的比较

| 模式 | 处理者选择 | 顺序 | 终止条件 |
|------|------------|------|----------|
| **责任链** | 动态 | 链式 | 处理或链尾 |
| **命令** | 直接 | 无 | 执行 |
| **装饰器** | 全部 | 链式 | 全部执行 |

#### 与其他模式的联合使用
- **责任链 + 组合**：构建树形责任链
- **责任链 + 观察者**：观察者通知形成责任链

### 4.7 访问者模式 (Visitor)

#### 主要用途和作用

**核心目的**：表示一个作用于某对象结构中的各元素的操作，使在不改变各元素的类的前提下定义作用于这些元素的新操作。

**适用场景**：
- 对象结构相对稳定，但经常需要在此结构上定义新的操作
- 需要对一个对象结构中的对象进行很多不同且不相关的操作
- 需要避免相关类污染

```cpp
// 访问者接口
class Visitor;

// 元素接口
class Element {
public:
    virtual void accept(Visitor& v) = 0;
    virtual ~Element() = default;
};

// 具体元素
class ConcreteElementA : public Element {
public:
    void accept(Visitor& v) override;
    void operationA() {
        std::cout << "ElementA operation\n";
    }
};

class ConcreteElementB : public Element {
public:
    void accept(Visitor& v) override;
    void operationB() {
        std::cout << "ElementB operation\n";
    }
};

// 访问者
class Visitor {
public:
    virtual void visit(ConcreteElementA& e) = 0;
    virtual void visit(ConcreteElementB& e) = 0;
    virtual ~Visitor() = default;
};

// 具体访问者
class ConcreteVisitor : public Visitor {
public:
    void visit(ConcreteElementA& e) override {
        std::cout << "Visitor processing ";
        e.operationA();
    }

    void visit(ConcreteElementB& e) override {
        std::cout << "Visitor processing ";
        e.operationB();
    }
};

// 元素实现 accept
void ConcreteElementA::accept(Visitor& v) {
    v.visit(*this);
}

void ConcreteElementB::accept(Visitor& v) {
    v.visit(*this);
}
```

#### 与其他模式的比较

| 模式 | 目的 | 操作位置 | 数据结构 |
|------|------|----------|----------|
| **访问者** | 添加操作 | 访问者中 | 稳定 |
| **迭代器** | 遍历 | 迭代器中 | 稳定 |
| **组合** | 树形结构 | 元素中 | 可变 |

#### 与其他模式的联合使用
- **访问者 + 组合**：遍历组合结构
- **访问者 + 迭代器**：迭代器驱动访问

## 一、排序算法

### 1.1 快速排序 (Quick Sort)

#### 算法详细介绍

**核心思想**：分治法。选择一个基准元素(pivot)，将数组分为两部分：小于pivot的元素和大于pivot的元素，然后递归排序这两部分。

**算法步骤**：
1. 从数组中选择一个元素作为基准
2. 重新排列数组，所有小于基准的元素放在前面，所有大于基准的元素放在后面（分区操作）
3. 递归地对小于基准值的子数组、大于基准值的子数组进行排序

**图解**：
```
原始数组: [5, 2, 9, 1, 6, 3]

第一轮: pivot = 3
       [2, 1]  3  [5, 9, 6]
       ↑小于3    ↑基准  ↑大于3

继续递归...
```

**C++ 实现**：

```cpp
// 分区函数 - Hoare分区方案
template <typename RandomIt>
RandomIt partition(RandomIt first, RandomIt last) {
    auto pivot = *(last - 1);  // 选择最后一个元素作为基准
    auto i = first - 1;

    for (auto j = first; j < last - 1; ++j) {
        if (*j <= pivot) {
            ++i;
            std::iter_swap(i, j);
        }
    }
    std::iter_swap(i + 1, last - 1);
    return i + 1;
}

// 标准快速排序
template <typename RandomIt>
void quickSort(RandomIt first, RandomIt last) {
    if (first < last - 1) {
        auto pivot = partition(first, last);
        quickSort(first, pivot);
        quickSort(pivot + 1, last);
    }
}
```

**三路划分**（处理大量重复元素）：

```cpp
// 三路划分：分成 < pivot, = pivot, > pivot 三部分
template <typename RandomIt>
void quickSort3Way(RandomIt first, RandomIt last) {
    if (first >= last) return;

    auto pivot = *first;
    auto lt = first;      // [first, lt): < pivot
    auto gt = last - 1;   // (gt, last): > pivot
    auto i = first;       // [lt, i): == pivot

    while (i <= gt) {
        if (*i < pivot) {
            std::iter_swap(lt++, i++);
        } else if (*i > pivot) {
            std::iter_swap(i, gt--);
        } else {
            ++i;
        }
    }

    quickSort3Way(first, lt);
    quickSort3Way(gt + 1, last);
}
```

#### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|------------|------|
| 最坏 | O(n²) | 数组已排序或逆序，每次分区极度不平衡 |
| 平均 | O(n log n) | 每次分区大致平衡 |
| 最好 | O(n log n) | 每次分区恰好平衡 |
| 空间复杂度 | O(log n) | 递归栈深度 |

**稳定性**：不稳定（相同元素的相对位置可能改变）

#### 与同类算法比较

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 | 原地 | 特点 |
|------|----------|----------|------|------|------|------|
| **快排** | O(n log n) | O(n²) | O(log n) | ✗ | ✓ | 实际最快 |
| **归并** | O(n log n) | O(n log n) | O(n) | ✓ | ✗ | 稳定、保证性能 |
| **堆排** | O(n log n) | O(n log n) | O(1) | ✗ | ✓ | 性能保证 |
| **插入** | O(n²) | O(n²) | O(1) | ✓ | ✓ | 小数组快 |
| **希尔** | O(n^1.3) | O(n²) | O(1) | ✗ | ✓ | 改进插入 |

**对比结论**：
- **快速排序 vs 归并排序**：快排原地排序但最坏O(n²)，归并需要额外空间但保证O(n log n)
- **快速排序 vs 堆排序**：快排缓存友好更快，堆排序最坏情况更好但常数大
- **快速排序 vs 插入排序**：大数组用快排，小数组插入排序更快

#### 实际工程应用
- **std::sort** - 通常使用 IntroSort（快排+堆排+插入排序混合）
- **qsort** - C标准库的快速排序
- **并行快排** - 多线程加速

### 1.2 归并排序 (Merge Sort)

#### 算法详细介绍

**核心思想**：分治法。将数组分成两半，分别排序后合并。

**算法步骤**：
1. 找到数组中点，将数组分成两半
2. 递归排序左半部分
3. 递归排序右半部分
4. 合并两个已排序的半部分

**图解**：
```
原始: [5, 2, 9, 1, 6, 3]

分解:
[5, 2, 9, 1, 6, 3]
    ↓
[5, 2, 9]  [1, 6, 3]
    ↓           ↓
[5] [2, 9]  [1] [6, 3]
    ↓    ↓      ↓   ↓
[2, 5] [9]  [1] [3, 6]
    ↓    ↓      ↓   ↓
[2, 5, 9]  [1, 3, 6]
       ↓
[1, 2, 3, 5, 6, 9]
```

**C++ 实现**：

```cpp
// 合并两个已排序区间
template <typename RandomIt>
void merge(RandomIt first, RandomIt mid, RandomIt last) {
    using T = typename std::iterator_traits<RandomIt>::value_type;
    std::vector<T> temp;
    auto left = first, right = mid;

    while (left < mid && right < last) {
        if (*left <= *right) {
            temp.push_back(std::move(*left++));
        } else {
            temp.push_back(std::move(*right++));
        }
    }

    temp.insert(temp.end(), left, mid);
    temp.insert(temp.end(), right, last);
    std::move(temp.begin(), temp.end(), first);
}

// 标准归并排序
template <typename RandomIt>
void mergeSort(RandomIt first, RandomIt last) {
    if (last - first <= 1) return;

    auto mid = first + (last - first) / 2;
    mergeSort(first, mid);
    mergeSort(mid, last);
    merge(first, mid, last);
}
```

**自然归并**（利用已排序的子数组）：

```cpp
// 查找已排序的run
template <typename RandomIt>
RandomIt findEndOfRun(RandomIt first, RandomIt last) {
    if (first == last) return first;
    auto next = first + 1;
    while (next < last && *next >= *(next - 1)) ++next;
    return next;
}

// 自然归并排序
template <typename RandomIt>
void naturalMergeSort(RandomIt first, RandomIt last) {
    if (last - first <= 1) return;

    std::vector<std::pair<RandomIt, RandomIt>> runs;
    auto it = first;
    while (it < last) {
        auto runEnd = findEndOfRun(it, last);
        runs.emplace_back(it, runEnd);
        it = runEnd;
    }

    // 逐轮合并相邻runs
    while (runs.size() > 1) {
        std::vector<std::pair<RandomIt, RandomIt>> newRuns;
        for (size_t i = 0; i < runs.size(); i += 2) {
            if (i + 1 < runs.size()) {
                merge(runs[i].first, runs[i].second, runs[i + 1].second);
                newRuns.emplace_back(runs[i].first, runs[i + 1].second);
            } else {
                newRuns.push_back(runs[i]);
            }
        }
        runs = std::move(newRuns);
    }
}
```

#### 复杂度分析

| 情况 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 最坏 | O(n log n) | O(n) |
| 平均 | O(n log n) | O(n) |
| 最好 | O(n log n) | O(n) |

**稳定性**：稳定

#### 与同类算法比较

归并排序 vs 快速排序 vs 堆排序：

| 特性 | 归并排序 | 快速排序 | 堆排序 |
|------|----------|----------|--------|
| 时间复杂度 | 总是 O(n log n) | 平均 O(n log n)，最坏 O(n²) | 总是 O(n log n) |
| 空间复杂度 | O(n) | O(log n) | O(1) |
| 稳定性 | 稳定 | 不稳定 | 不稳定 |
| 原地排序 | 否 | 是 | 是 |
| 缓存友好度 | 中等 | 高 | 低 |
| 适合外部排序 | ✓ | ✗ | ✗ |
| 适合链表 | ✓ | ✗ | ✗ |

**对比结论**：
- 选择归并排序当：需要稳定性、处理大数据、外部排序、链表排序
- 选择快速排序当：平均性能最重要、内存受限
- 选择堆排序当：保证最坏情况性能、内存严格受限

### 1.3 堆排序 (Heap Sort)

#### 算法详细介绍

**核心思想**：利用堆数据结构进行排序。堆是一个完全二叉树，满足堆性质：父节点的值总是大于（或小于）子节点。

**算法步骤**：
1. 将数组构建成最大堆
2. 取出堆顶（最大值）与堆的最后一个元素交换
3. 调整剩余元素使其满足堆性质
4. 重复2-3直到堆为空

**图解**：
```
建堆:
      9
     / \
    5   6
   / \ /
  2  1 3

[9, 5, 6, 2, 1, 3]

排序过程:
1. 交换9与3: [3, 5, 6, 2, 1, | 9]
2. 调整:    [6, 5, 3, 2, 1, | 9]
3. 交换6与1: [1, 5, 3, 2, | 6, 9]
4. 调整:    [5, 2, 3, 1, | 6, 9]
...
```

**C++ 实现**：

```cpp
// 堆化调整
template <typename RandomIt, typename Compare>
void heapify(RandomIt first, RandomIt last, RandomIt root, Compare comp) {
    auto len = last - first;
    auto largest = root;
    auto left = first + (root - first) * 2 + 1;
    auto right = left + 1;

    if (left < last && comp(*largest, *left)) {
        largest = left;
    }
    if (right < last && comp(*largest, *right)) {
        largest = right;
    }

    if (largest != root) {
        std::iter_swap(root, largest);
        heapify(first, last, largest, comp);
    }
}

// 堆排序
template <typename RandomIt, typename Compare = std::less<>>
void heapSort(RandomIt first, RandomIt last, Compare comp = {}) {
    // 建堆（从最后一个非叶子节点开始）
    auto len = last - first;
    for (auto it = first + len / 2 - 1; it >= first; --it) {
        heapify(first, last, it, comp);
    }

    // 排序
    for (auto end = last - 1; end > first; --end) {
        std::iter_swap(first, end);  // 将最大值移到末尾
        heapify(first, end, first, comp);  // 调整堆
    }
}
```

#### 复杂度分析

| 情况 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 最坏 | O(n log n) | O(1) |
| 平均 | O(n log n) | O(1) |
| 最好 | O(n log n) | O(1) |

**稳定性**：不稳定

#### 与同类算法比较

堆排序 vs 快速排序 vs 归并排序：

| 特性 | 堆排序 | 快速排序 | 归并排序 |
|------|--------|----------|----------|
| **最坏时间** | O(n log n) ✓ | O(n²) ✗ | O(n log n) ✓ |
| **空间** | O(1) ✓ | O(log n) | O(n) ✗ |
| **缓存** | 差 ✗ | 好 ✓ | 中 |
| **稳定** | 否 ✗ | 否 ✗ | 是 ✓ |
| **原地** | 是 ✓ | 是 ✓ | 否 ✗ |

**对比结论**：
- 堆排序适合：内存严格受限、对最坏情况有要求
- 不适合：追求性能、缓存敏感场景

### 1.4 插入排序

#### 算法详细介绍

**核心思想**：将未排序元素插入到已排序部分的正确位置。

**算法步骤**：
1. 第一个元素视为已排序
2. 取出下一个元素，在已排序序列中从后向前扫描
3. 如果已排序元素大于新元素，将该元素向后移动
4. 重复步骤3直到找到正确位置，插入新元素

**C++ 实现**：

```cpp
template <typename RandomIt>
void insertionSort(RandomIt first, RandomIt last) {
    for (auto it = first + 1; it < last; ++it) {
        auto key = *it;
        auto j = it - 1;

        while (j >= first && *j > key) {
            *(j + 1) = *j;  // 向后移动
            --j;
        }
        *(j + 1) = key;
    }
}
```

#### 复杂度分析

| 情况 | 时间复杂度 |
|------|------------|
| 最坏 | O(n²) - 逆序 |
| 平均 | O(n²) |
| 最好 | O(n) - 已排序 |

#### 与同类算法比较

| 算法 | 小数组 | 已排序 | 逆序 | 稳定 |
|------|--------|--------|------|------|
| **插入** | 快 | O(n) | O(n²) | 是 |
| **选择** | 慢 | O(n²) | O(n²) | 否 |
| **冒泡** | 慢 | O(n) | O(n²) | 是 |

### 1.5 希尔排序

#### 算法详细介绍

**核心思想**：改进的插入排序，先对间隔较大的元素进行插入排序，逐步缩小间隔。

**C++ 实现**：

```cpp
template <typename RandomIt>
void shellSort(RandomIt first, RandomIt last) {
    auto n = last - first;
    for (auto gap = n / 2; gap > 0; gap /= 2) {
        for (auto i = gap; i < n; ++i) {
            auto temp = *(first + i);
            auto j = i;

            while (j >= gap && *(first + j - gap) > temp) {
                *(first + j) = *(first + j - gap);
                j -= gap;
            }
            *(first + j) = temp;
        }
    }
}
```

#### 复杂度分析

| 间隔序列 | 最坏时间复杂度 |

| Hibbard (2^k-1) | O(n^1.5) |
| Sedgewick | O(n^1.33) |
| 最好情况 | O(n log n) |

### 1.6 计数排序

#### 算法详细介绍

**核心思想**：统计每个元素出现的次数，按顺序输出。非比较排序，适合整数排序。

**C++ 实现**：

```cpp
void countingSort(const std::vector<int>& arr) {
    if (arr.empty()) return;

    int max = *std::max_element(arr.begin(), arr.end());
    int min = *std::min_element(arr.begin(), arr.end());

    std::vector<int> count(max - min + 1, 0);

    // 统计
    for (int num : arr) {
        count[num - min]++;
    }

    // 输出
    std::vector<int> result;
    for (size_t i = 0; i < count.size(); ++i) {
        for (int j = 0; j < count[i]; ++j) {
            result.push_back(i + min);
        }
    }
    return result;
}
```

| 最好 | O(n + k) | O(k) |
| 最坏 | O(n + k) | O(k) |
| 平均 | O(n + k) | O(k) |

其中 k = max - min 是数值范围。

## 二、查找算法

### 2.1 二分查找

#### 算法详细介绍

**核心思想**：在有序数组中，每次将搜索范围减半。

**算法步骤**：
1. 设 left = 0, right = n - 1
2. 如果 left > right，查找失败
3. mid = (left + right) / 2
4. 如果 arr[mid] == target，找到
5. 如果 arr[mid] < target，在右半部分继续
6. 如果 arr[mid] > target，在左半部分继续

**图解**：
```
数组: [1, 3, 5, 7, 9, 11, 13], target = 7

第1轮: left=0, right=6, mid=3, arr[3]=7 → 找到！

如果 target = 6:
第1轮: left=0, right=6, mid=3, arr[3]=7 > 6
       → left=0, right=2
第2轮: left=0, right=2, mid=1, arr[1]=3 < 6
       → left=2, right=2
第3轮: left=2, right=2, mid=2, arr[2]=5 < 6
       → left=3, right=2
       → left > right, 未找到
```

**C++ 实现**：

```cpp
// 标准二分查找
template <typename T, typename U>
constexpr T* binarySearch(T* first, T* last, const U& value) {
    auto len = last - first;
    while (len > 0) {
        auto half = len / 2;
        auto mid = first + half;

        if (*mid < value) {
            first = mid + 1;
            len = len - half - 1;
        } else if (*mid > value) {
            len = half;
        } else {
            return mid;  // 找到
        }
    }
    return nullptr;  // 未找到
}

// lower_bound: 第一个 >= value 的位置
template <typename T, typename U>
constexpr T* lowerBound(T* first, T* last, const U& value) {
    auto len = last - first;
    while (len > 0) {
        auto half = len / 2;
        auto mid = first + half;

        if (*mid < value) {
            first = mid + 1;
            len = len - half - 1;
        } else {
            len = half;
        }
    }
    return first;
}

// upper_bound: 第一个 > value 的位置
template <typename T, typename U>
constexpr T* upperBound(T* first, T* last, const U& value) {
    auto len = last - first;
    while (len > 0) {
        auto half = len / 2;
        auto mid = first + half;

        if (!(value < *mid)) {  // *mid <= value
            first = mid + 1;
            len = len - half - 1;
        } else {
            len = half;
        }
    }
    return first;
}

// 二分查找旋转数组
int searchRotated(const std::vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) return mid;

        // 判断哪半边有序
        if (nums[left] <= nums[mid]) {  // 左半边有序
            if (nums[left] <= target && target < nums[mid]) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        } else {  // 右半边有序
            if (nums[mid] < target && target <= nums[right]) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
    }
    return -1;
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 查找 | O(log n) | O(1) |
| 插入 | O(n) | O(n) |
| 删除 | O(n) | O(n) |

#### 与同类算法比较

| 查找算法 | 时间复杂度 | 是否有序 | 数据结构 |
|----------|-------------|----------|----------|
| **二分查找** | O(log n) | 是 | 数组 |
| **顺序查找** | O(n) | 否 | 任意 |
| **哈希查找** | O(1)平均 | 否 | 哈希表 |
| **树查找** | O(log n) | 否 | BST |

**对比结论**：
- 二分查找：有序数组的最佳选择
- 哈希查找：更快但需要额外空间
- 树查找：适合动态数据

### 2.2 哈希查找

#### 算法详细介绍

**核心思想**：通过哈希函数将键映射到存储位置，实现O(1)查找。

**哈希函数设计**：

```cpp
// 乘法哈希
template <typename T>
size_t multiplicativeHash(T key, size_t m) {
    const double A = 0.6180339887;  // (sqrt(5) - 1) / 2
    double val = key * A;
    val -= std::floor(val);
    return static_cast<size_t>(m * val);
}

// FNV-1a 哈希
size_t fnv1aHash(const std::string& str) {
    const size_t FNV_prime = 1099511628211ULL;
    const size_t FNV_offset_basis = 14695981039346656037ULL;
    size_t hash = FNV_offset_basis;

    for (char c : str) {
        hash ^= static_cast<size_t>(c);
        hash *= FNV_prime;
    }
    return hash;
}
```

**冲突解决**：

```cpp
// 开放寻址法（线性探测）
template <typename K, typename V, size_t N = 1024>
class HashMap {
    enum class State { EMPTY, OCCUPIED, DELETED };

    struct Entry {
        K key;
        V value;
        State state = State::EMPTY;
    };

    std::array<Entry, N> table;

    size_t hash(const K& key) const {
        return std::hash<K>{}(key) % N;
    }

    size_t probe(size_t index, size_t i) const {
        return (index + i) % N;
    }

public:
    bool insert(const K& key, const V& value) {
        size_t index = hash(key);

        for (size_t i = 0; i < N; ++i) {
            size_t pos = probe(index, i);

            if (table[pos].state == State::EMPTY ||
                table[pos].state == State::DELETED) {
                table[pos].key = key;
                table[pos].value = value;
                table[pos].state = State::OCCUPIED;
                return true;
            }

            if (table[pos].state == State::OCCUPIED &&
                table[pos].key == key) {
                table[pos].value = value;  // 更新
                return true;
            }
        }
        return false;  // 表已满
    }

    bool find(const K& key, V& value) const {
        size_t index = hash(key);

        for (size_t i = 0; i < N; ++i) {
            size_t pos = probe(index, i);

            if (table[pos].state == State::EMPTY) {
                return false;
            }

            if (table[pos].state == State::OCCUPIED &&
                table[pos].key == key) {
                value = table[pos].value;
                return true;
            }
        }
        return false;
    }
};
```

#### 复杂度分析

| 冲突解决 | 平均查找 | 成功查找 | 失败查找 |
|----------|----------|----------|----------|
| 开放寻址 | O(1) | ~1 + α/2 | ~1/(1-α)/2 |
| 链地址法 | O(1) | 1 + α/2 | α + e^(-α) |

其中 α = 装载因子 = 元素数/表大小

#### 与同类算法比较

| 查找算法 | 平均 | 最坏 | 有序要求 | 额外空间 |
|----------|------|------|----------|----------|
| **哈希** | O(1) | O(n) | 否 | O(n) |
| **二分** | O(log n) | O(log n) | 是 | O(1) |
| **顺序** | O(n) | O(n) | 否 | O(1) |

## 三、图算法

### 3.1 最短路径

#### 3.1.1 Dijkstra 算法

**算法详细介绍**：

**核心思想**：贪心策略。每次选择距离起点最近的未访问节点，更新其邻居的距离。

**适用场景**：
- 边权非负的有向/无向图
- 单源最短路径

**图解**：
```
图: A--4-->B--2-->D
     |       |       ^
     2       3       |
     v       v       |
     C--1-->D--1-->E

从A出发到各点的最短距离:
A: 0
B: 4 (A→B)
C: 2 (A→C)
D: 3 (A→C→D)
E: 4 (A→C→D→E)
```

**C++ 实现**：

```cpp
#include <queue>
#include <unordered_map>
#include <vector>
using Edge = std::pair<int, int>;  // (weight, destination)

std::unordered_map<int, int> dijkstra(
    const std::unordered_map<int, std::vector<Edge>>& graph,
    int start
) {
    std::unordered_map<int, int> dist;
    using P = std::pair<int, int>;  // (distance, vertex)
    std::priority_queue<P, std::vector<P>, std::greater<>> pq;

    pq.emplace(0, start);
    dist[start] = 0;

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (d > dist[u]) continue;  // 已找到更短路径

        for (const auto& [weight, v] : graph.at(u)) {
            int newDist = dist[u] + weight;
            if (!dist.count(v) || newDist < dist[v]) {
                dist[v] = newDist;
                pq.emplace(newDist, v);
            }
        }
    }
    return dist;
}
```

#### 复杂度分析

| 数据结构 | 时间复杂度 | 空间复杂度 |
|----------|-------------|-------------|
| 二叉堆 | O((V + E) log V) | O(V) |
| 斐波那契堆 | O(V log V + E) | O(V) |
| 数组 | O(V²) | O(V) |

#### 3.1.2 Bellman-Ford 算法

**算法详细介绍**：

**核心思想**：动态规划。对所有边进行 V-1 轮松弛操作。

**适用场景**：
- 可以处理负权边
- 检测负权环

```cpp
std::unordered_map<int, int> bellmanFord(
    const std::vector<std::tuple<int, int, int>>& edges,  // (u, v, weight)
    int start,
    int V
) {
    const int INF = 1e9;
    std::unordered_map<int, int> dist;

    // 初始化
    dist[start] = 0;
    for (int i = 1; i < V; ++i) {
        for (const auto& [u, v, w] : edges) {
            if (dist.count(u) && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
            }
        }
    }

    // 检测负权环
    for (const auto& [u, v, w] : edges) {
        if (dist.count(u) && dist[u] + w < dist[v]) {
            throw std::runtime_error("Negative cycle detected");
        }
    }

    return dist;
}
```

#### 复杂度分析

| 情况 | 时间复杂度 |
|------|-------------|
| 一般 | O(V·E) |
| 最坏 | O(V·E) |
| 空间复杂度 | O(V) |

#### 3.1.3 Floyd-Warshall 算法

**算法详细介绍**：

**核心思想**：动态规划。dp[i][j][k] = min(dp[i][j][k-1], dp[i][k][k-1] + dp[k][j][k-1])

**适用场景**：
- 全源最短路径
- 可以检测负权环
- 密集图

```cpp
const int INF = 1e9;

void floydWarshall(std::vector<std::vector<int>>& dist) {
    int n = dist.size();

    // 初始化
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < n; ++j) {
            if (i != j && dist[i][j] == 0) {
                dist[i][j] = INF;
            }
        }
    }

    // 动态规划
    for (int k = 0; k < n; ++k) {
        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < n; ++j) {
                if (dist[i][k] + dist[k][j] < dist[i][j]) {
                    dist[i][j] = dist[i][k] + dist[k][j];
                }
            }
        }
    }
}
```

#### 复杂度分析

| 复杂度类型 | 值 |
|------------|-----|
| 时间 | O(V³) |
| 空间 | O(V²) |

#### 最短路径算法对比

| 算法 | 时间复杂度 | 适用场景 | 负权边 | 负权环 |
|------|-------------|----------|--------|--------|
| **Dijkstra** | O((V+E)log V) | 非负权单源 | ✗ | - |
| **Bellman-Ford** | O(V·E) | 通用单源 | ✓ | ✓(检测) |
| **Floyd-Warshall** | O(V³) | 全源、密集图 | ✓ | ✓(检测) |
| **SPFA** | O(k·E) 平均 | 通用单源 | ✓ | ✓(检测) |

**选择指南**：
- 单源、非负权 → Dijkstra
- 单源、可能有负权 → Bellman-Ford
- 全源、密集图 → Floyd-Warshall
- 稀疏图全源 → 多次 Dijkstra

### 3.2 最小生成树 (MST)

#### 3.2.1 Prim 算法

**算法详细介绍**：

**核心思想**：从任意节点开始，每次添加与当前树相连的最小边。

**图解**：
```
图:     2      3
    A ---- B ---- D
    |      |      |
   4|      1|     4
    |      |      |
    C ---- E ---- F
         1      2

Prim 从 A 开始:
1. 加入 A, 最小边 AB=2
2. 加入 B, 最小边 BE=1
3. 加入 E, 最小边 EC=1
4. 加入 C, 最小边 EA=2(已选) → 选 EC=1(已选) → 选 AB=2(已选)
5. 加入 D, 最小边 BD=3
6. 加入 F, 最小边 EF=2

MST 总权值: 2+1+1+3+2 = 9
```

**C++ 实现**：

```cpp
#include <queue>

int prim(const std::vector<std::vector<std::pair<int, int>>>& graph) {
    // graph[u] = [(v, w), ...]
    int n = graph.size();
    std::vector<bool> visited(n, false);
    std::priority_queue<std::pair<int, int>,
                        std::vector<std::pair<int, int>>,
                        std::greater<>> pq;

    pq.emplace(0, 0);  // (weight, vertex)
    int totalWeight = 0;

    while (!pq.empty()) {
        auto [weight, u] = pq.top();
        pq.pop();

        if (visited[u]) continue;
        visited[u] = true;
        totalWeight += weight;

        for (const auto& [v, w] : graph[u]) {
            if (!visited[v]) {
                pq.emplace(w, v);
            }
        }
    }
    return totalWeight;
}
```

#### 复杂度分析

| 数据结构 | 时间复杂度 | 空间复杂度 |
|----------|-------------|-------------|
| 二叉堆 | O(E log V) | O(V) |
| 斐波那契堆 | O(E + V log V) | O(V) |

#### 3.2.2 Kruskal 算法

**算法详细介绍**：

**核心思想**：按边权排序，依次选择不形成环的边。

**C++ 实现**：

```cpp
class UnionFind {
    std::vector<int> parent, rank;

public:
    UnionFind(int n) : parent(n), rank(n, 0) {
        std::iota(parent.begin(), parent.end(), 0);
    }

    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);  // 路径压缩
        }
        return parent[x];
    }

    bool unite(int x, int y) {
        x = find(x);
        y = find(y);
        if (x == y) return false;

        if (rank[x] < rank[y]) std::swap(x, y);
        parent[y] = x;
        if (rank[x] == rank[y]) rank[x]++;
        return true;
    }
};

int kruskal(std::vector<std::tuple<int, int, int>> edges, int n) {
    // edges: (weight, u, v)
    std::sort(edges.begin(), edges.end());
    UnionFind uf(n);
    int totalWeight = 0;

    for (const auto& [w, u, v] : edges) {
        if (uf.unite(u, v)) {
            totalWeight += w;
        }
    }
    return totalWeight;
}
```

#### 复杂度分析

| 操作 | 时间复杂度 |
|------|-------------|
| 排序边 | O(E log E) |
| 并查集操作 | O(E α(V)) ≈ O(E) |
| 总时间 | O(E log E) = O(E log V) |
| 空间 | O(V) |

#### MST 算法对比

| 算法 | 时间复杂度 | 适用场景 |
|------|-------------|----------|
| **Prim** | O(E log V) | 稠疏图 |
| **Prim(邻接矩阵)** | O(V²) | 密集图 |
| **Kruskal** | O(E log E) | 稀疏图、边已排序 |

## 四、动态规划

### 4.1 背包问题

#### 算法详细介绍

**0-1 背包**：每个物品只能选一次
**完全背包**：每个物品可以选多次

**C++ 实现**：

```cpp
// 0-1 背包
int knapsack01(const std::vector<int>& weights,
               const std::vector<int>& values,
               int capacity) {
    int n = weights.size();
    std::vector<int> dp(capacity + 1, 0);

    for (int i = 0; i < n; ++i) {
        for (int j = capacity; j >= weights[i]; --j) {
            dp[j] = std::max(dp[j], dp[j - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}

// 完全背包
int knapsackComplete(const std::vector<int>& weights,
                     const std::vector<int>& values,
                     int capacity) {
    int n = weights.size();
    std::vector<int> dp(capacity + 1, 0);

    for (int i = 0; i < n; ++i) {
        for (int j = weights[i]; j <= capacity; ++j) {
            dp[j] = std::max(dp[j], dp[j - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
```

#### 复杂度分析

| 类型 | 时间 | 空间 |
|------|------|------|
| 0-1 背包 | O(n·W) | O(W) |
| 完全背包 | O(n·W) | O(W) |

### 4.2 最长递增子序列 (LIS)

#### 算法详细介绍

**核心思想**：dp[i] = 以 i 结尾的最长递增子序列长度

**C++ 实现**：

```cpp
// O(n²) DP
int lisDP(const std::vector<int>& nums) {
    if (nums.empty()) return 0;
    int n = nums.size();
    std::vector<int> dp(n, 1);

    for (int i = 1; i < n; ++i) {
        for (int j = 0; j < i; ++j) {
            if (nums[j] < nums[i]) {
                dp[i] = std::max(dp[i], dp[j] + 1);
            }
        }
    }
    return *std::max_element(dp.begin(), dp.end());
}

// O(n log n) 二分查找
int lisBinarySearch(const std::vector<int>& nums) {
    std::vector<int> tails;

    for (int num : nums) {
        auto it = std::lower_bound(tails.begin(), tails.end(), num);
        if (it == tails.end()) {
            tails.push_back(num);
        } else {
            *it = num;
        }
    }
    return tails.size();
}
```

#### 复杂度分析

| 算法 | 时间 | 空间 |
|------|------|------|
| DP | O(n²) | O(n) |
| 二分查找 | O(n log n) | O(n) |

## 五、数据结构

### 5.1 并查集

#### 算法详细介绍

**核心思想**：用树结构维护集合，支持合并和查询操作。

**C++ 实现**：

```cpp
class DSU {
    std::vector<int> parent, rank;

public:
    DSU(int n) : parent(n), rank(n, 0) {
        std::iota(parent.begin(), parent.end(), 0);
    }

    // 查找根节点（带路径压缩）
    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }

    // 合并两个集合（按秩合并）
    bool unite(int x, int y) {
        x = find(x);
        y = find(y);
        if (x == y) return false;

        if (rank[x] < rank[y]) std::swap(x, y);
        parent[y] = x;
        if (rank[x] == rank[y]) rank[x]++;
        return true;
    }

    bool connected(int x, int y) {
        return find(x) == find(y);
    }
};
```

#### 复杂度分析

| 优化 | 查找 | 合并 |
|------|------|------|
| 无优化 | O(n) | O(1) |
| 路径压缩 | O(α(n)) | O(1) |
| 按秩合并 | O(log n) | O(α(n)) |
| 两者结合 | O(α(n)) | O(α(n)) |

其中 α(n) 是反阿克曼函数，增长极慢， practically < 5。

### 5.2 线段树

#### 算法详细介绍

**核心思想**：用二叉树维护区间信息，支持区间查询和更新。

**C++ 实现**：

```cpp
class SegmentTree {
    std::vector<int> tree;
    int n;

    void build(const std::vector<int>& arr, int node, int start, int end) {
        if (start == end) {
            tree[node] = arr[start];
        } else {
            int mid = (start + end) / 2;
            build(arr, 2 * node, start, mid);
            build(arr, 2 * node + 1, mid + 1, end);
            tree[node] = tree[2 * node] + tree[2 * node + 1];
        }
    }

public:
    SegmentTree(const std::vector<int>& arr) {
        n = arr.size();
        tree.resize(4 * n);
        build(arr, 1, 0, n - 1);
    }

    int query(int l, int r) {
        return query(1, 0, n - 1, l, r);
    }

    void update(int idx, int val) {
        update(1, 0, n - 1, idx, val);
    }
};
```

#### 复杂度分析

| 操作 | 时间 | 空间 |
|------|------|------|
| 构建 | O(n) | O(n) |
| 查询 | O(log n) | O(1) |
| 更新 | O(log n) | O(1) |
